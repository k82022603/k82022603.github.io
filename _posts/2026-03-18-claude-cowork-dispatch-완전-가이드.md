## 1. 개요: Dispatch란 무엇인가?

**Dispatch**는 Anthropic이 2026년 3월 17일 발표한 Claude Cowork의 신기능으로, 현재 리서치 프리뷰(Research Preview) 단계에 있습니다. 한 문장으로 설명하자면 이렇습니다.

> *"스마트폰으로 지시를 내리면, 집에 있는 데스크탑 컴퓨터의 Claude가 그 일을 대신 처리해 드립니다."*

기존 Claude는 대화창에서 요청을 받아 텍스트 응답을 돌려주는 방식이었습니다. Dispatch는 이 개념을 완전히 뒤집습니다. 사용자는 외출 중에 스마트폰으로 작업 지시를 내리고, Claude는 사용자의 데스크탑에서 로컬 파일, 연결된 서비스, 브라우저에 접근하며 실제 작업을 수행합니다. 사용자가 돌아왔을 때는 완성된 결과물이 기다리고 있습니다.

이 기능은 Claude Code의 Remote Control 기능과 유사한 설계 철학을 공유하지만, 터미널 없이 사용할 수 있는 일반 사용자 친화적인 버전입니다.

## 2. 핵심 작동 원리

Dispatch의 작동 방식은 크게 세 가지 구조로 이루어져 있습니다.

### 2.1 하나의 지속적인 대화 스레드 (One Persistent Conversation)

Dispatch는 단순히 모바일과 데스크탑을 연결하는 것이 아닙니다. 두 기기 사이에서 **하나의 연속된 대화**가 동기화됩니다. 스마트폰에서 보낸 메시지, Claude의 응답, 작업 결과물이 모두 같은 대화 창에 쌓입니다. 대화 도중 자리를 비워도 Claude는 혼자 작업을 계속하고, 돌아왔을 때 결과를 확인할 수 있습니다.

### 2.2 데스크탑 실행 구조

Claude는 사용자의 **데스크탑 컴퓨터 위에서** 실행됩니다. 스마트폰이 Claude에게 명령을 전달하면, Claude는 데스크탑의 Claude 앱을 통해 작업을 처리합니다. 이를 통해 다음이 가능해집니다.

- 로컬 파일 시스템에 직접 접근 (다운로드 폴더, 문서 폴더 등)
- 연결된 외부 서비스(Slack, Gmail, Google Drive 등)와 상호작용
- 사용자가 설치한 플러그인 및 MCP 앱 활용
- 브라우저 제어 및 웹 검색 수행

### 2.3 샌드박스 환경에서의 코드 실행

Cowork의 특성상, Claude는 격리된 가상 머신(VM) 환경에서 코드를 실행합니다. 이는 보안과 안정성을 위한 설계이며, 생성된 파일은 사용자가 지정한 폴더에 직접 저장됩니다.

## 3. 실제 시연 장면 분석 ([영상 기반](https://www.youtube.com/watch?v=fVIV-L49eBs))

Anthropic이 공개한 시연 영상과 스크린샷은 Dispatch의 실제 활용 시나리오를 생생하게 보여줍니다. 세 장의 이미지를 순서대로 분석합니다.

### 장면 1: 모바일에서 작업 지시

첫 번째 화면은 스마트폰에 표시된 **"Cowork dispatch"** 인터페이스입니다. 상단에는 초록색 점과 함께 "Online" 상태가 표시되어 있어 데스크탑과의 연결이 활성화되어 있음을 알 수 있습니다.

사용자는 다음과 같이 메시지를 입력합니다.

> *"Can you open the Acme landing page proposal in my downloads and give me the key points? I have like 15 min before I need to present it!"*
> (번역: "다운로드 폴더에 있는 Acme 랜딩 페이지 제안서를 열어서 핵심 포인트를 알려줄 수 있어? 발표까지 15분밖에 없어!")

이 메시지 하나로 Claude는 구체적인 지시 없이도 다음 작업들을 스스로 판단하고 실행합니다. 파일 위치 파악, 파일 열기, 내용 분석, 핵심 요약 생성의 과정이 자동으로 진행됩니다.

### 장면 2: 데스크탑에서 파일 탐색 중

두 번째 화면은 스마트폰과 데스크탑 화면이 동시에 보이는 장면입니다. 왼쪽 스마트폰 화면에서는 Claude가 `"On it — pulling up the deck now."` (지금 바로 찾아볼게요)라고 응답하며 실시간 작업 상태를 알려주고 있습니다. 화면 중앙에는 `"Locating file..."` 텍스트가 애니메이션처럼 표시되며, 오른쪽 데스크탑 화면에는 실제 macOS의 파인더(Finder) 창이 열려 있습니다.

파인더의 다운로드 폴더 안에는 다음 파일들이 보입니다.
- `Acme_Landing_Refresh.pptx` ← Claude가 찾는 바로 그 파일
- `Benefits_Open_Enrollment_2026.pdf`
- `brand_assets_export.zip`
- `expenses_march.csv`
- `invoice_8842.pdf`
- `Q1_forecast_v2.xlsx`
- 그 외 스크린샷, 항공권 확인서 등

Claude는 자동으로 파일을 탐색하고, 수많은 파일 중에서 사용자가 원하는 `Acme_Landing_Refresh.pptx`를 정확히 찾아냅니다.

### 장면 3: 분석 결과 및 심층 질의응답

세 번째 화면은 Claude가 프레젠테이션 파일을 분석한 후 제공한 결과입니다.

Claude는 먼저 발표에서 활용할 수 있는 핵심 포인트를 제시합니다.

> *"Lead with the 68% bounce rate, land on the $370K. The page speed claim (4.2s → sub-2s) is the one that'll get questions."*
> (번역: "68% 이탈률로 시작해서 $370K로 마무리하세요. 페이지 속도 주장(4.2초 → 2초 이하)은 질문이 들어올 부분입니다.")

사용자는 여기서 더 깊은 질문을 합니다.

> *"How do they get to 370K? Want to be able to back that up if someone asks."*
> (번역: "370K는 어떻게 계산된 거야? 누가 물어보면 대답할 수 있어야 해서.")

Claude는 프레젠테이션 내 차트를 분석하여 상세한 근거를 설명합니다.

> *"The stacked bar chart. Baseline revenue stays roughly flat ($320–350K/quarter), the green incremental layer grows from +$28K in Q1 to +$168K in Q4. Sum the green bars and you get ~$373K. If anyone pushes..."*
> (번역: "누적 막대 차트 기준입니다. 기준 매출은 분기당 $320~350K로 거의 변동이 없고, 초록색 증분 레이어가 Q1의 +$28K에서 Q4의 +$168K로 성장합니다. 초록색 막대를 모두 합산하면 약 $373K가 됩니다.")

이 장면은 Dispatch가 단순한 파일 검색 도구를 넘어, 내용을 깊이 이해하고 사용자의 맥락에 맞는 인사이트를 제공하는 **대화형 분석 파트너**임을 보여줍니다.

## 4. 시작하기: 설치 및 페어링

### 4.1 필수 요건

Dispatch를 사용하려면 다음 두 가지가 모두 필요합니다.

**데스크탑 환경**
- Claude Desktop 앱 최신 버전 (macOS 또는 Windows x64)
- Windows ARM64는 현재 미지원
- 앱이 실행 중인 상태에서 컴퓨터가 깨어있어야 함

**모바일 환경**
- Claude 모바일 앱 최신 버전 (iOS / Android)
- 기존 앱 사용자는 최신 버전으로 업데이트 필수

앱 다운로드: [claude.com/download](https://claude.com/download)

### 4.2 페어링 과정

1. Claude Desktop 앱을 최신 버전으로 업데이트합니다.
2. Claude Desktop의 Cowork 탭을 열면 왼쪽 패널에 **"Dispatch"** 항목이 새로 생깁니다.
3. Dispatch를 클릭하면 기능 설명 화면이 나타납니다.
4. **"Get started"** 버튼을 클릭합니다.
5. 다음 화면에서 두 가지 옵션을 설정할 수 있습니다.
   - **파일 접근 권한 허용**: Claude가 로컬 파일에 접근할 수 있도록 허용
   - **컴퓨터 절전 방지 유지**: 작업 중 컴퓨터가 절전 모드로 전환되지 않도록 설정
6. QR 코드가 화면에 표시됩니다.
7. 스마트폰의 Claude 앱을 열고 QR 코드를 스캔하여 페어링을 완료합니다.
8. 이제 스마트폰의 **"Dispatch"** 섹션에서 메시지를 보내면 됩니다.

페어링 후에는 두 기기 간 대화가 자동으로 동기화됩니다.

## 7. 현재 한계점 및 주의사항

Dispatch는 리서치 프리뷰 단계이므로 다음과 같은 제한이 있습니다.

### 7.1 기술적 제한

**데스크탑 활성화 필수**: Claude는 데스크탑에서 실행됩니다. 컴퓨터가 절전 모드이거나 Claude Desktop 앱이 닫혀있으면 작업이 불가능합니다.

**단일 스레드 구조**: 현재 새로운 대화를 시작하거나 여러 개의 스레드를 관리하는 기능이 없습니다. 모든 메시지는 하나의 연속된 대화에 쌓입니다.

**작업 완료 알림 없음**: 현재 모바일이나 데스크탑 모두에서 작업 완료 알림(Push Notification)이 제공되지 않습니다. 직접 앱을 열어서 확인해야 합니다.

**예약 작업 불가**: 예약 작업(Scheduled Tasks)은 별도로 관리되며, 이 Dispatch 스레드 내에서는 지원되지 않습니다.

**속도 및 안정성**: 초기 테스트에 따르면 작업이 느리고, 성공률이 약 50% 수준으로 보고되고 있습니다. 아직 중요한 업무에 전적으로 의존하기에는 이른 단계입니다.

**일방향 통신**: Claude는 메시지에만 반응합니다. 작업이 완료되어도 먼저 알림을 보내거나 적극적으로 연락을 취하지 않습니다.

**세션 메모리 없음**: Cowork의 특성상 세션 간 메모리가 유지되지 않습니다.

### 7.2 플랫폼 제한

- 중간에 Claude Desktop 오프라인 전환 시 작업이 중단될 수 있습니다.
- Cowork 활동은 감사 로그(Audit Logs), 규정 준수 API, 데이터 내보내기에 기록되지 않습니다.
- 규제가 필요한 업무에는 사용하지 않도록 권장됩니다.

## 8. 보안 및 안전 고려사항

Dispatch는 강력한 만큼 신중한 사용이 필요합니다. Anthropic은 다음과 같은 보안 원칙을 명시하고 있습니다.

### 8.1 에이전트 체인의 위험성

스마트폰의 모바일 AI 에이전트가 데스크탑 AI 에이전트를 원격으로 제어하는 구조입니다. 이는 스마트폰의 지시가 데스크탑에서 실제 행동으로 이어지는 체인을 만들어냅니다. 이 과정에서 다음과 같은 위험이 발생할 수 있습니다.

- 로컬 파일 읽기, 이동, 삭제
- 연결된 외부 서비스와의 상호작용
- 브라우저 제어

### 8.2 프롬프트 인젝션 위험

Claude가 작업 중 접근하는 악성 파일, 웹페이지, 또는 조작된 문서에 포함된 명령이 예기치 않은 행동을 유발할 수 있습니다. 이는 취소하기 어려운 결과로 이어질 수 있습니다.

### 8.3 권장 사항

- Cowork에서 Claude에게 어떤 접근 권한을 부여했는지 항상 인지하고 있어야 합니다.
- 의도한 행동뿐만 아니라, Claude가 잠재적으로 할 수 있는 모든 행동이 허용 범위 내에 있는지 확인한 후 연결하세요.
- 규제가 필요한 중요한 업무에는 사용하지 마세요.
- 공식 가이드: [Use Cowork Safely](https://support.claude.com/en/articles/13947068)

## 9. Claude Cowork란? (배경 이해)

Dispatch를 이해하려면 먼저 **Claude Cowork**를 이해해야 합니다.

Cowork는 Claude Code의 에이전트 기능을 터미널 없이 사용할 수 있도록 설계된 Claude Desktop의 고급 기능입니다. 단순히 채팅에 응답하는 것이 아니라, 복잡한 다단계 작업을 자율적으로 수행합니다.

### Cowork의 주요 특징

**직접 파일 접근**: 사용자가 수동으로 파일을 업로드하거나 다운로드할 필요 없이, Claude가 로컬 파일에 직접 읽고 씁니다.

**실제 결과물 생성**: 텍스트 응답이 아니라, PowerPoint, Excel, Word, PDF 같은 실제 파일을 지정된 폴더에 저장합니다.

**병렬 작업 처리**: 복잡한 작업을 여러 서브태스크로 나누고 병렬로 처리합니다.

**예약 작업**: 특정 시간에 자동으로 작업을 수행할 수 있습니다(Dispatch 스레드 외).

**서브 에이전트 조율**: 여러 워크스트림을 동시에 조율합니다.

Dispatch는 이 강력한 Cowork 기능을 스마트폰에서도 접근할 수 있도록 확장한 것입니다.

## 10. 자주 묻는 질문 (FAQ)

**Q. Dispatch를 사용하려면 항상 컴퓨터가 켜져있어야 하나요?**  
네. Claude는 사용자의 데스크탑 컴퓨터에서 실행되므로, 컴퓨터가 켜져있고 Claude Desktop 앱이 실행 중이어야 합니다. 이를 위해 "컴퓨터 절전 방지" 설정을 활성화하는 것을 권장합니다.

**Q. 스마트폰에서만 사용할 수 있나요?**  
아닙니다. 스마트폰과 데스크탑 모두에서 Dispatch 대화에 접근할 수 있습니다. 핵심은 "이동 중에 스마트폰으로 지시를 내릴 수 있다"는 점입니다.

**Q. 어떤 파일 형식을 처리할 수 있나요?**  
Cowork를 통해 PowerPoint(.pptx), Excel(.xlsx), Word(.docx), PDF, CSV, 이미지 등 다양한 형식을 처리할 수 있습니다.

**Q. Dispatch와 일반 Claude 채팅의 차이는 무엇인가요?**  
일반 Claude 채팅은 텍스트 응답을 제공하며 로컬 파일에 직접 접근할 수 없습니다. Dispatch(Cowork)는 실제 로컬 파일 시스템, 연결된 서비스, 브라우저에 접근하여 실제 파일 결과물을 생성합니다.

**Q. 작업이 얼마나 걸리나요?**  
현재 리서치 프리뷰 단계에서는 작업이 다소 느릴 수 있습니다. 복잡도에 따라 수분에서 수십 분이 소요될 수 있으며, 초기 사용자 리뷰에 따르면 성공률이 약 50% 수준입니다.

**Q. 어떤 외부 서비스와 연동되나요?**  
Cowork의 Connector 기능을 통해 Slack, Gmail, Google Drive, Figma, Asana, Box, Canva, Monday.com 등 다양한 MCP 앱과 연동할 수 있습니다.

## 마치며

Claude Cowork의 Dispatch 기능은 단순한 편의 기능이 아닙니다. "AI가 백그라운드에서 알아서 일을 처리해준다"는 개념이 일반 사용자에게 처음으로 현실화되는 순간입니다. 발표 15분 전에 스마트폰으로 보낸 한 줄의 메시지가 수십 페이지 분량의 프레젠테이션을 분석해 핵심을 짚어주는 경험, 그것이 Dispatch가 보여주는 미래입니다.

현재는 리서치 프리뷰로 속도와 안정성에 한계가 있지만, Anthropic은 지속적인 업데이트를 예고하고 있습니다. 지금이야말로 이 기능을 직접 체험하며 업무 흐름에 어떻게 통합할지 고민해볼 최적의 타이밍입니다.
