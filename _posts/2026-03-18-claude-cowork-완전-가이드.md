## 1. Claude Cowork란 무엇인가

Claude Cowork는 Anthropic이 2026년 1월 출시한 에이전틱(Agentic) 데스크탑 도구로, Claude 데스크탑 앱에 내장되어 있습니다. 단순한 챗봇 인터페이스를 넘어서, 사용자의 컴퓨터 파일에 직접 읽기·쓰기 권한을 갖고 복잡한 작업을 자율적으로 수행할 수 있는 **작업 실행 에이전트**입니다.

Cowork의 핵심 철학은 "대화"가 아닌 "작업 위임"입니다. 사용자는 원하는 결과물을 설명하고 자리를 떠날 수 있으며, 돌아왔을 때 완성된 파일이 지정 폴더에 저장되어 있습니다.

### 핵심 작동 방식

사용자가 작업을 요청하면, Cowork는 다음 과정을 자율적으로 수행합니다.

첫째, 요청을 분석하고 실행 계획을 수립합니다. 둘째, 복잡한 작업을 여러 하위 작업으로 분해합니다. 셋째, 컴퓨터의 가상 머신(VM) 환경에서 하위 작업들을 병렬 또는 순차적으로 실행합니다. 넷째, 완성된 결과물을 지정된 폴더에 직접 저장합니다. 이 전 과정에서 사용자의 개입은 최소화되거나 불필요합니다.

### 주요 기능 목록

- **직접 파일 접근**: 수동 업로드 없이 로컬 파일에 직접 읽기·쓰기 가능
- **서브 에이전트 조율**: 복잡한 작업을 병렬 워크스트림으로 처리
- **스킬(Skills)**: 반복 가능한 AI 워크플로우 저장 및 재사용
- **커넥터(Connectors)**: Gmail, Google Drive, Slack, Apollo 등 38개 이상의 외부 앱 연동
- **플러그인(Plugins)**: 스킬과 커넥터의 묶음으로 특정 역할에 최적화된 AI 에이전트
- **예약 작업(Scheduled Tasks)**: `/schedule` 명령으로 정해진 시간에 자동 실행

## 2. Claude Chat · Claude Code · Cowork 비교

세 가지 도구는 같은 Claude 모델을 기반으로 하지만, 목적과 사용 방식이 명확히 다릅니다. 어떤 도구를 언제 사용해야 하는지 이해하는 것이 효율적인 활용의 첫걸음입니다.

### Claude Chat (대화 에이전트)

Claude Chat은 프롬프트-응답 방식의 대화형 AI입니다. 사용자가 항상 루프 안에 있으며, 질문에 대한 답변을 즉시 받을 수 있습니다. 웹 검색, 아티팩트 생성, 프로젝트 기반 컨텍스트 유지가 주요 기능입니다. 짧은 질문, 브레인스토밍, 초안 작성처럼 5~10분 내에 끝나는 작업에 가장 적합합니다.

### Claude Code (개발자 에이전트)

Claude Code는 개발자를 위한 자율적 코딩 도구입니다. 터미널에서 실행되며, 코드 작성·실행, Git 저장소 관리, 명령어 실행 등 기술적 작업을 담당합니다. 기술적 배경이 없는 사용자에게는 진입 장벽이 높지만, 코드를 이해하는 개발자에게는 매우 강력한 도구입니다. Cowork가 Claude Code와 동일한 아키텍처 위에 구축되어 있기 때문에, 기술적으로는 Cowork에서도 Claude Code가 할 수 있는 대부분의 작업이 가능합니다.

### Claude Cowork (업무 에이전트)

Cowork는 비기술적 사용자(Non-technical operators)를 위한 자율 실행 도구입니다. Claude Code와 동일한 자율 실행 능력을 갖추고 있지만, 터미널 대신 파일과 폴더 기반의 시각적 인터페이스로 작동합니다. 비즈니스 운영자, 콘텐츠 크리에이터, 영업 담당자 등 일상적인 지식 업무를 자동화하고 싶은 모든 사람에게 적합합니다.

아래 비교표는 세 도구의 핵심 차이를 정리한 것입니다.

| 구분 | Claude Chat | Claude Code | Cowork |
|---|---|---|---|
| **성격** | 대화형 어시스턴트 | 개발자 도구 | 업무 실행 에이전트 |
| **인터페이스** | 웹/앱 채팅 | 터미널 | 데스크탑 앱 (시각적) |
| **자율성** | 낮음 (항상 루프인) | 높음 | 높음 |
| **파일 접근** | 업로드 방식 | 직접 접근 | 직접 접근 |
| **기억 방식** | Projects | CLAUDE.md, Skills | Skills |
| **주요 작업** | Q&A, 초안 작성 | 앱 개발, 배포 | 파일 처리, 업무 자동화 |
| **대상 사용자** | 일반인 | 개발자 | 비즈니스 운영자 |

한 마디로 요약하면, Claude Chat은 질문에 답하는 어시스턴트, Claude Code는 소프트웨어를 만드는 개발자, Cowork는 작업을 완수하는 직원입니다.

## 3. 초기 설정 가이드

### 필요한 플랜

Cowork는 무료 플랜에서는 사용할 수 없습니다. Pro, Max, Team, Enterprise 플랜 중 하나가 필요합니다. 처음 시작한다면 Pro 플랜(월 $20)을 권장합니다. 사용량 한도에 자주 부딪힌다면 Max 플랜으로 업그레이드를 고려하세요. Pro 대비 Max는 사용량 한도가 5배 더 높습니다.

Cowork는 복잡한 멀티스텝 작업을 처리하므로 일반 채팅보다 토큰 소모가 많습니다. 이 점을 감안하여 관련 작업을 하나의 세션으로 묶어 처리하는 것이 효율적입니다.

### 데스크탑 앱 다운로드

Cowork는 현재 데스크탑 앱 전용 기능입니다. 브라우저나 모바일에서는 사용할 수 없습니다.

1. [claude.com/download](https://claude.com/download) 접속
2. 본인의 운영체제에 맞는 버전 선택 (macOS 또는 Windows)
3. 앱 설치 후 Claude 계정으로 로그인
4. 사이드바에서 **Cowork** 탭 클릭

> **참고**: 2026년 2월부터 Windows도 macOS와 동일한 기능 수준을 지원합니다. 단, Windows arm64는 현재 지원되지 않습니다.

### 첫 번째 작업 폴더 설정

데스크탑 앱을 열면 입력창 하단에 **"Work in a folder"** 드롭다운이 보입니다.

1. 드롭다운을 클릭하고 "Choose a different folder" 선택
2. 원하는 위치에 새 폴더 생성 (예: `Claude-Workspace`)
3. 해당 폴더를 선택하면 접근 권한 요청 팝업이 나타남
4. **"Always Allow"** 클릭하여 영구 권한 부여

Claude는 이 폴더와 하위 폴더에 있는 모든 파일을 읽고, 수정하고, 새로 만들 수 있습니다. 처음 설정 시에는 중요한 파일의 백업본을 별도 위치에 보관하는 것을 권장합니다.

## 4. 워크스페이스 구조 설계

단순히 폴더 하나를 연결하는 것보다, 처음부터 체계적인 구조를 설계하면 장기적으로 훨씬 효율적으로 활용할 수 있습니다. 영상의 진행자 Bart는 다음과 같은 구조를 권장합니다.

### 권장 폴더 구조

```
Claude-Workspace/
├── context/          ← 나에 관한 상시 참조 파일 모음
│   ├── about-me.md
│   ├── brand-voice.md
│   └── working-preferences.md
├── projects/         ← 현재 진행 중인 프로젝트들
│   ├── project-a/
│   └── project-b/
└── outputs/          ← Claude가 완성하여 납품하는 결과물
```

이 구조를 Claude에게 직접 만들어달라고 요청할 수 있습니다. "빈 폴더에 접근 권한을 드렸습니다. context, projects, outputs 하위 폴더를 만들어 주세요"와 같이 지시하면, Claude가 폴더를 생성하고 각 폴더의 역할을 설명하는 README.md 파일까지 함께 작성해 줍니다.

### 핵심 MD 파일 3종 세트

워크스페이스에서 가장 중요한 것은 컨텍스트 파일입니다. Claude는 새 작업을 시작할 때마다 이 파일들을 읽어 사용자에 대한 배경 지식을 채웁니다. 아래 세 가지 MD 파일을 `context/` 폴더에 구성하는 것을 권장합니다.

**about-me.md**: 나는 누구인가, 어떤 일을 하는가, 현재 우선순위는 무엇인가를 담은 파일입니다. Claude가 모든 작업의 맥락을 이해하는 기반이 됩니다.

**brand-voice.md**: 내가 어떻게 소통하는가를 정의하는 파일입니다. 선호하는 어조와 표현, 피해야 할 표현, 실제 작성 글 예시 등을 포함합니다. Claude가 생성하는 콘텐츠가 자신의 목소리처럼 들리게 만드는 핵심 파일입니다.

**working-preferences.md**: Claude와의 협업 방식을 정의하는 파일입니다. 새 프로젝트 시작 방법, 작업 완료 후 저장 위치, 로그 기록 방식 등 워크플로우 전반에 대한 선호도를 담습니다.

이 파일들을 처음에 Claude와 대화를 통해 작성하면 좋습니다. "about-me.md, brand-voice.md, working-preferences.md 파일을 만들어 주세요. 각 파일에 필요한 정보를 수집하기 위해 여러 질문을 해주세요"라고 요청하면 Claude가 체계적으로 질문을 던지고 파일을 작성해 줍니다.

## 5. 실전 활용 예시: 인보이스 처리

영상에서 가장 인상적으로 소개된 예시는 다수의 인보이스 파일을 자동 처리하는 시나리오입니다. 이 예시는 Cowork의 핵심 능력을 잘 보여줍니다.

### 시나리오

데스크탑에 여러 장의 인보이스 이미지 파일이 무작위로 저장되어 있습니다. 이것을 카테고리별로 하위 폴더에 정리하고, 모든 인보이스 데이터를 엑셀 스프레드시트로 정리하고 싶습니다.

### 실행 과정

프롬프트를 다음과 같이 작성합니다.

> "첨부 폴더에 있는 인보이스들을 카테고리별로 하위 폴더에 정리해 주세요. 그리고 모든 인보이스를 카테고리, 소계(subtotal), 세금(tax), 합계(total)로 구분한 엑셀 시트도 만들어 주세요."

이 단순한 프롬프트를 받은 Cowork는 다음을 자율적으로 수행합니다.

첫째, 폴더 안의 인보이스 파일들을 모두 읽고 내용을 파악합니다. 둘째, 카테고리를 자동으로 분류합니다(예: 식음료, 연료·교통 등). 셋째, 카테고리별 하위 폴더를 생성하고 파일을 이동합니다. 넷째, 각 인보이스의 핵심 데이터를 추출하여 엑셀 파일에 정리합니다. 다섯째, 카테고리 합계와 전체 합계를 자동으로 계산합니다.

영상에서 실제로 처리된 결과를 보면, 5개의 인보이스가 "Food & Dining"과 "Fuel & Travel" 두 카테고리로 정확히 분류되었으며, 각 항목별 소계·세금·합계가 엑셀 파일에 정리되었습니다.

### Claude가 엑셀 수식을 사용하는 이유

Cowork는 합계 값을 직접 숫자로 입력하는 것이 아니라 엑셀 수식(예: `=SUM(B2:B6)`)으로 작성합니다. 이렇게 하면 AI가 계산 결과를 잘못 기억하거나 추정하는 "환각(Hallucination)" 현상이 원천적으로 방지됩니다. 수식이 계산하므로 수치는 항상 정확합니다.

## 6. 스킬(Skills) 이해와 활용

### 스킬이란?

스킬은 반복 가능한 AI 워크플로우를 저장하고 재사용할 수 있게 해주는 기능입니다. 한 번 수행한 작업 과정을 스킬로 저장해 두면, 다음에는 단 하나의 명령어(슬래시 커맨드)로 동일한 작업을 즉시 실행할 수 있습니다.

### 스킬 만드는 방법

가장 효과적인 스킬 생성 방법은 실제로 작업을 먼저 완수하고 나서, 그 작업을 스킬로 변환하도록 요청하는 것입니다.

인보이스 처리 예시를 다시 사용하면, 작업 완료 후 다음과 같이 요청합니다.

> "이 과정을 재사용 가능한 스킬로 만들어 주세요. `/summarize-invoices` 명령어로 실행되도록 하고, 폴더의 새로운 인보이스를 확인해서 카테고리 정리 후 엑셀에 기록하는 스킬로 만들어 주세요."

Claude는 스킬 파일을 생성하고, 이를 스킬 탭에 설치해 줍니다. 이후에는 `/summarize-invoices`만 입력하면 매번 상세한 지시 없이도 동일한 과정이 자동 실행됩니다.

### 토큰 절약 전략

Cowork는 파일을 읽을 때마다 토큰을 소비합니다. 인보이스가 100개라면 모두 읽는 데 상당한 토큰이 사용됩니다. 스킬에 스크립트를 내장하면 이 문제를 해결할 수 있습니다. Claude Code 기반으로 파싱 스크립트를 작성하고, 스킬이 그 스크립트를 실행하도록 구성하면 매 실행 시 토큰 소비를 크게 줄일 수 있습니다.

### 스킬 버전 2.0 업데이트

최근 스킬 기능이 업데이트되어, 단순히 프로세스를 저장하는 것을 넘어 평가 기준(criteria)을 설정하고 결과를 검증할 수 있게 되었습니다. 사용자가 스킬 검토 과정에 참여하여 최적화 후 배포하는 방식으로, 더 높은 품질과 일관성을 보장합니다.

---

## 7. 커넥터(Connectors)와 외부 앱 연동

### 커넥터란?

커넥터는 Cowork가 외부 앱과 상호작용할 수 있게 해주는 연동 기능입니다. 현재 38개 이상의 커넥터가 지원됩니다.

**기본 제공 커넥터**: Gmail, Google Calendar, Google Drive, GitHub, Slack, Notion, Canva, Excalidraw, Miro, ClickUp, Claude in Chrome(브라우저 자동화), PowerPoint(Anthropic 제공)

**추가 연동 가능 커넥터**: Apollo.io, Salesforce, Supabase, Asana 등

### 커넥터 연결 방법

1. 사이드바에서 **Customize** → **Connectors** 진입
2. 연결하려는 앱 선택 후 **Connect** 클릭
3. OAuth 인증 흐름에 따라 앱에 로그인하여 권한 부여
4. 연결 완료 후 **읽기/쓰기 권한을 세부 조정** 가능

예를 들어 Apollo.io 커넥터에서는 연락처 검색(Read), 시퀀스에 연락처 추가(Write) 등의 각 기능별로 "Always Allow", "Needs Approval", "Blocked" 중 하나를 선택할 수 있습니다.

### Claude in Chrome 활용

Claude in Chrome은 Chrome 브라우저 익스텐션으로, Cowork와 연동하면 로컬 파일 처리와 웹 기반 워크플로우를 결합할 수 있습니다. 버튼 클릭, 양식 작성, 탭 간 이동 등을 자동화할 수 있으며, Gmail, Google Docs, Slack, GitHub 등 주요 플랫폼에 이미 최적화되어 있습니다.

---

## 8. 플러그인(Plugins) 생태계

### 플러그인이란?

플러그인은 스킬, 커넥터, 슬래시 커맨드, 서브 에이전트를 하나의 패키지로 묶은 것입니다. 특정 역할이나 부서에 맞게 Claude를 전문화된 AI 직원으로 만들어주는 기능입니다.

비유하자면, 신입 직원을 고용하면 회사의 플레이북(업무 프로세스와 도구 목록)을 전달하듯, 플러그인은 Claude에게 특정 역할의 플레이북을 설치하는 것입니다.

### Anthropic 공식 플러그인

Anthropic은 2026년 1월 플러그인 마켓플레이스를 출시하며 자체적으로 사용하는 11개의 플러그인을 오픈소스로 공개했습니다. 플러그인 종류로는 마케팅, 세일즈, 리서치, 생산성, 법무, 재무 등 다양한 부서 특화형이 있습니다.

### Apollo 플러그인 예시

영상에서 소개된 Apollo 플러그인은 세 가지 스킬을 포함합니다.

- `/enrich-lead`: 리드 데이터 보강
- `/prospect`: 신규 잠재 고객 탐색
- `/sequence-load`: 이메일 시퀀스에 연락처 추가

이 플러그인 하나로 "새 리드 찾기 → 데이터 보강 → 시퀀스 이메일 발송" 전체 영업 프로세스를 자동화할 수 있습니다.

### 플러그인 설치 방법

1. **Customize** → **Browse plugins** 진입
2. 원하는 플러그인 검색 및 선택
3. **Install** 클릭
4. 설치 후 커넥터 권한 확인 및 필요에 따라 커스터마이징

플러그인은 `Customize` 버튼으로 자신의 도구 스택, 회사 용어, 프로세스에 맞게 수정할 수 있습니다.

## 9. 예약 작업(Scheduled Tasks) 설정

### 예약 작업이란?

예약 작업은 Cowork가 사용자의 개입 없이 특정 시간에 자동으로 작업을 실행하도록 설정하는 기능입니다. 스킬과 결합하면 완전히 자율적인 업무 자동화가 가능합니다.

### 예약 작업 설정 방법

작업창에서 `/schedule`을 입력하거나, 사이드바의 **Scheduled** 탭에서 직접 설정할 수 있습니다.

인보이스 예시를 계속 사용하면, 스킬 설치 후 다음과 같이 요청합니다.

> "이 스킬을 매주 월요일 오전 9시에 실행되는 예약 작업으로 만들어 주세요."

Claude는 즉시 예약 작업을 구성하고, 실행 일정과 상세 실행 계획을 보여줍니다. 승인을 하면 설정이 완료됩니다.

실제 영상에서 구성된 예약 작업 예시는 다음과 같습니다.

- **Daily email summary**: 평일 오전 9:01 — Gmail 받은편지함 요약
- **Monday invoice summary**: 매주 월요일 오전 9:08 — 새 인보이스 처리 및 엑셀 업데이트

### 중요한 제약사항

예약 작업은 Claude 데스크탑 앱이 열려 있고 컴퓨터가 켜져 있을 때만 실행됩니다. 앱이 닫혀 있거나 컴퓨터가 절전 모드이면 실행되지 않습니다. 앱 설정에서 "Keep awake" 옵션을 켜면 컴퓨터가 절전 상태로 전환되지 않도록 할 수 있지만, 불필요한 전력 소비가 발생할 수 있으므로 신중하게 사용하세요.

---

## 10. 전역 설정(Global Instructions) 최적화

### 전역 설정의 중요성

전역 설정은 모든 Cowork 세션에서 Claude가 반드시 읽고 따르는 기본 지시사항입니다. 이 설정을 잘 구성하면 매 작업마다 같은 설명을 반복하지 않아도 됩니다.

### 접근 방법

데스크탑 앱 → **Settings** → **Cowork** → 지시사항 입력란에 작성

### 권장 전역 설정 예시

영상에서 소개된 효과적인 전역 설정 예시입니다.

```
저는 [이름]입니다.

새 작업을 시작하기 전에 항상 컨텍스트 파일을 먼저 읽어 주세요.
- 실행 전에 항상 명확화 질문을 먼저 해주세요.
- 실행 전에 간단한 계획을 먼저 보여주세요.
- 출력물 기본 형식: .docx
- 파일을 명시적 승인 없이 삭제하지 마세요.
```

가장 중요한 것은 **"작업 시작 전에 컨텍스트 파일을 읽을 것"** 이라는 지시입니다. 이 한 줄이 Claude의 망각 문제를 크게 줄이고 일관성을 높여줍니다.

### 폴더별 설정(Folder Instructions)

전역 설정 외에도 특정 폴더에 진입할 때만 적용되는 폴더별 설정도 지원됩니다. 예를 들어 `projects/youtube/` 폴더에는 유튜브 콘텐츠 제작 관련 지시사항을, `projects/sales/` 폴더에는 영업 관련 지시사항을 따로 설정할 수 있습니다.

---

## 11. 서브 에이전트(Sub-agents)와 병렬 처리

Cowork는 복잡한 작업을 처리할 때 여러 서브 에이전트를 병렬로 실행할 수 있습니다. 이 기능 덕분에 단일 에이전트가 순차적으로 처리하는 것보다 훨씬 많은 양의 작업을 단시간에 완수할 수 있습니다.

예를 들어 스킬 생성 작업을 수행할 때, Claude는 스킬 파일 작성, 테스트 케이스 설계, 문서화 등을 여러 서브 에이전트에게 분산하여 동시에 처리합니다. 진행 상황은 우측 Progress 패널에서 실시간으로 확인할 수 있습니다.

서브 에이전트 기능은 플러그인과 결합될 때 특히 강력합니다. 하나의 요청으로 다수의 고객 이메일 초안 작성, CRM 데이터 업데이트, 캘린더 일정 생성 등 여러 작업을 동시에 처리할 수 있습니다.

## 12. 활용 팁과 주의사항

### 효과적인 활용 팁

**작업을 구체적으로 지시하되 결과 중심으로 설명하세요.** "이 일을 해주세요"가 아니라 "이런 결과물을 원합니다"처럼 원하는 최종 상태를 명확히 기술하는 것이 좋습니다.

**Claude 데스크탑 앱을 열어두세요.** 앱을 닫으면 진행 중인 세션이 종료됩니다. 장시간 작업이나 예약 작업은 앱이 열려 있어야 합니다.

**출처 추적을 요청하세요.** 여러 파일을 종합하는 작업에서 "각 내용이 어느 파일에서 왔는지 파일명을 표시해 주세요"라고 요청하면 나중에 검증이 쉬워집니다.

**토큰 사용량을 모니터링하세요.** Settings → Usage에서 확인 가능합니다. 관련 작업은 단일 세션으로 묶어 처리하면 효율적입니다.

**기존 AI 대화 컨텍스트를 활용하세요.** ChatGPT 등에서 쌓은 대화 기록이나 컨텍스트를 내보내 워크스페이스 폴더에 저장하면, Cowork가 그 내용을 읽어 이어서 작업할 수 있습니다.

### 주의사항

**중요 파일은 반드시 백업을 유지하세요.** Cowork는 파일을 직접 수정할 수 있으므로, 실수로 덮어쓰거나 삭제될 위험이 있습니다. 전역 설정에 "명시적 승인 없이 파일 삭제 금지"를 반드시 포함하세요.

**복잡한 엑셀 파일 파싱은 아직 제한적입니다.** 머지된 셀, 복잡한 멀티 영역 레이아웃을 가진 스프레드시트는 파싱 오류가 발생할 수 있습니다.

**Chrome 자동화는 스크린샷 기반으로 작동하여 속도가 다소 느릴 수 있습니다.**

**규제된 워크로드에는 사용을 피하세요.** Cowork 활동은 현재 감사 로그, 컴플라이언스 API에 포함되지 않습니다.

**작업이 중간에 멈췄다면** 컴퓨터가 절전 모드로 전환되었거나 앱이 종료된 경우입니다. 앱을 다시 열고 작업 기록에서 중단 지점을 확인한 후 이어서 요청하면 됩니다.

## 템플릿 3: working-preferences.md

```markdown
# Working Preferences

## 기본 작업 규칙
- 작업 시작 전에 항상 context 폴더의 MD 파일들을 먼저 읽어주세요.
- 실행 전에 짧은 계획(Plan)을 먼저 제시하고 승인을 받아주세요.
- 모호한 부분이 있으면 실행 전에 명확화 질문을 해주세요.
- 파일을 삭제하기 전에는 반드시 명시적 확인을 받아주세요.

## 출력물 형식
- 문서: [예: .docx 기본 / 마크다운이 필요한 경우 .md]
- 스프레드시트: .xlsx
- 이미지/자료: outputs/ 폴더에 저장
- 파일명 규칙: [예: YYYY-MM-DD_작업명 형식. 예: 2026-03-18_뉴스레터초안.docx]

## 프로젝트 관리 방식

### 새 프로젝트 시작 시
1. projects/ 폴더 안에 프로젝트명으로 하위 폴더 생성
2. 해당 폴더에 README.md 파일 작성 (목표, 마감, 담당자 등)
3. 작업 진행 상황을 README.md에 체크리스트로 업데이트

### 작업 완료 시
1. 완성된 결과물은 outputs/ 폴더로 이동
2. 프로젝트 README.md에 완료 날짜와 최종 파일 경로 기록
3. 작업 요약 한 문단을 README.md 하단에 추가

## 반복 작업 목록 (스킬로 전환 대상)
| 작업명 | 주기 | 슬래시 커맨드 |
|---|---|---|
| [예: 주간 이메일 요약] | [매주 월요일] | [/weekly-email-summary] |
| [예: 인보이스 정리] | [매주 금요일] | [/sort-invoices] |
| [예: SNS 콘텐츠 초안] | [격주] | [/draft-social-content] |

## 커뮤니케이션 선호도
- 결과물 요약: 항상 작업 완료 후 3줄 이내 요약을 제공해주세요.
- 오류 발생 시: 무엇이 잘못되었는지, 어떻게 해결했는지 간략히 설명해주세요.
- 불확실한 경우: 추측으로 진행하지 말고 먼저 질문해주세요.
- 언어: 기본 한국어, 파일명과 코드는 영어

## 보안 및 민감 정보
- [예: 고객 이름이 포함된 파일은 outputs/confidential/ 폴더에 별도 저장]
- [예: API 키, 비밀번호는 절대 파일에 직접 기록하지 않음]
```
