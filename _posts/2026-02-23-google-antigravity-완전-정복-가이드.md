## 1. Google Antigravity란 무엇인가

Google Antigravity는 2025년 11월 18일 Gemini 3 모델 패밀리 출시와 함께 공개된 **에이전트 퍼스트(Agent-First) AI IDE**다. VS Code를 기반으로 하며, Google이 24억 달러에 인수한 Windsurf 팀이 주도하여 개발했다. 현재 무료 퍼블릭 프리뷰 상태로 Windows, macOS, Linux 모두 지원된다.

기존 AI 코딩 어시스턴트(GitHub Copilot, Cursor 등)가 자동완성이나 코드 제안에 초점을 맞췄다면, Antigravity는 **자율 AI 에이전트가 프로젝트 계획, 코드 작성, 테스트, 디버깅, 배포까지 전체 워크플로우를 처리**하는 방식으로 완전히 다른 패러다임을 제시한다. 

Sergey Brin이 직접 "Founder Mode"로 참여하여 에이전트 기능을 정교화한 것으로도 유명하며, 그의 목표는 Google을 "검색(Search)"에서 "실행(Action)"으로 전환하는 것이었다.

**지원 모델:**
- Gemini 3 Pro (기본, 무료 티어에서 관대한 속도 제한 제공)
- Gemini 3 Deep Think (심층 계획 및 복잡한 작업용)
- Claude Sonnet 4.5 / Claude Opus 4.5 (Anthropic)
- GPT-OSS-120B (OpenAI 오픈소스 변형)

이 가이드에서는 Jack Roberts가 수백 시간의 실전 경험으로 정리한 **6가지 인피니티 스톤(핵심 전략)**을 통해 Antigravity를 완전히 정복하는 방법을 다룬다.

## 2. 레벨 1: 셋업과 프로젝트 조직화

올바른 셋업이 모든 것의 출발점이다. 처음 설정을 잘못하면 나중에 성능 저하와 비효율이 누적된다.

### 2.1 프로젝트 단위 기준: "1 URL = 1 프로젝트"

가장 흔한 실수는 하나의 프로젝트 폴더에 너무 많은 것을 몰아넣는 것이다. 단순한 원칙을 따르자.

> **하나의 URL = 하나의 프로젝트 폴더**

대시보드와 랜딩 페이지가 서로 다른 URL을 갖는다면, 각각 별도의 프로젝트로 관리해야 한다. Antigravity의 Agent Manager 기능을 활용하면 언제든지 개별 프로젝트와 자유롭게 대화할 수 있어 작업 전환 비용이 크게 줄어든다.

### 2.2 Antigravity Skills 폴더 구성

Skills 폴더는 Antigravity의 전문 지식 패키지를 담는 중앙 저장소다. 구글 코드랩 공식 문서에 따르면, Skills는 **프로그레시브 디스클로저(Progressive Disclosure)** 방식으로 작동한다. 즉, 에이전트가 특정 요청을 감지할 때만 해당 스킬을 컨텍스트 윈도우에 로드하므로, 불필요한 토큰 소모 없이 전문 기능을 선택적으로 활용할 수 있다.

**Skills 폴더 초기 설정 방법:**
1. 데스크탑에서 `File → Open Folder → New Folder` 클릭
2. 폴더명을 `antigravity-skills`로 지정
3. 이후 모든 신규 프로젝트에 이 폴더를 임포트

**스킬의 두 가지 범위:**
- **글로벌 스코프** (`~/.gemini/antigravity/skills/`): 모든 프로젝트에서 사용 가능한 범용 스킬 (예: JSON 포맷팅, 일반 코드 리뷰)
- **워크스페이스 스코프** (`<workspace-root>/.agent/skills/`): 특정 프로젝트에서만 사용하는 스킬 (예: 특정 스테이징 서버 배포, 팀 고유 보일러플레이트 생성)

### 2.3 확장(Extensions) 탭 활용

왼쪽 사이드바의 Extensions 탭은 크롬 확장 프로그램과 유사한 개념이다. `claude code`를 검색하여 설치하면 Antigravity 내에서 Claude를 직접 활용할 수 있다. 시각화 도구나 코드 프레젠테이션 방식을 변경하는 다양한 확장이 있으므로 주기적으로 살펴보는 것이 좋다.

### 2.4 글로벌 규칙(Rules)과 워크플로우(Workflows) 설정

우측 상단의 점 세 개(⋮) → `Customizations`에서 두 가지 강력한 기능에 접근할 수 있다.

**Rules (전역 규칙):**
모든 대화에 자동으로 추가되는 하드코딩된 지침이다. 컨텍스트 윈도우를 소비하지만, 개인 코딩 스타일, 특정 프레임워크 선호도, 디자인 컬러 팔레트 등 변하지 않는 선호 사항을 여기에 기록해두면 매번 설명할 필요가 없다.

```
예시 Rules 내용:
- 모든 Python 코드는 타입 힌트를 포함할 것
- UI 디자인 시 검정/흰색/녹색 팔레트 사용
- 데이터베이스는 항상 Supabase 사용
```

**Workflows (워크플로우):**
특정 상황에서만 호출되는 마크다운 파일이다. 대화창에서 `/`를 입력하면 등록된 워크플로우 목록이 나타나며, 필요할 때만 로드되므로 컨텍스트 낭비가 없다.

```
예시 활용:
/workflow-debugging  → 디버깅 절차 자동 실행
/deploy-vercel       → Vercel 배포 단계 실행
/code-review         → 코드 리뷰 체크리스트 실행
```

### 2.5 GitHub 버전 관리 연동

프로젝트 진행 중 정기적으로 GitHub에 커밋하는 습관이 중요하다. Antigravity에게 "변경 사항을 GitHub에 푸시하고 완료되면 확인해줘"라고 요청하면 에이전트가 자동으로 처리한다. 무언가 잘못되었을 때 원하는 시점으로 즉시 롤백할 수 있는 것이 핵심 이점이다.

### 2.6 BLAST 마스터 프레임워크

Jack Roberts가 수십 개의 프로젝트 경험을 바탕으로 개발한 프로젝트 초기화 프롬프트 체계다. 새 프로젝트를 시작할 때 이 프레임워크를 붙여넣으면 에이전트가 단계적으로 안내한다.

**BLAST 프레임워크 5단계:**

| 단계 | 명칭 | 내용 |
|------|------|------|
| Phase 0 | 초기화 프로토콜 | 작업 계획(목표/체크리스트), 발견 사항(연구/제약), 진행 상황(완료/오류/테스트) 문서 생성 |
| Phase 1 | 블루프린트 | 비전과 로직, 통합 요소, 진실의 원천(데이터 위치), 전달 페이로드, 행동 규칙 정의 |
| Phase 2 | 링크(연결) | 필요한 외부 연결(Supabase, ClickUp 등), 핸드셰이크 검증 |
| Phase 3 | 아키텍처 레이어 | MVP(최소 기능 제품) 구축을 위한 3단계 프로세스 |
| Phase 4 | 스타일리제이션 | UI 정제 및 디자인 승인 |
| Phase 5 | 트리거 | 실행 환경 설정 (클라우드 자동화 등) |

프레임워크 초기화 시 에이전트는 `task_plan.md`, `progress.md`, `gemini.mmd`(프로젝트 헌법) 파일을 자동 생성한다.

## 3. 레벨 2: 성능 극대화 전략

설정이 완료되었다면 이제 Antigravity를 최고 성능으로 끌어올릴 차례다.

### 3.1 컨텍스트 오염(Context Rot) 방지

컨텍스트 윈도우는 Claude Opus 4.6 기준 100만 토큰으로 매우 크지만, **대화가 절반을 넘어서면 성능이 점진적으로 저하**된다. 이를 "컨텍스트 오염"이라고 부른다.

**해결책:** 하나의 작업이 완료되면 반드시 `+` 버튼을 눌러 새 대화창을 시작하라. 오른쪽 사이드바에서 언제든지 이전 대화로 되돌아갈 수 있으므로 맥락을 잃을 걱정은 없다.

### 3.2 3가지 에이전트 접근 방식

Antigravity 내에서 에이전트와 소통하는 방법은 세 가지다:

**방법 1: 기본 에이전트 채팅 패널**
사이드바의 기본 대화창. 모든 프로젝트 파일에 접근 가능하며 가장 통합된 경험을 제공한다.

**방법 2: Claude 플러그인 패널**
에디터 중앙 패널에서 더블클릭하면 Claude 플러그인이 나타난다. 에이전트가 오작동하거나 응답이 없을 때 대체 수단으로 활용한다. 무한히 새 인스턴스를 생성할 수 있으며, 우클릭으로 화면 배치를 자유롭게 조정 가능하다.

**방법 3: 터미널에서 직접 호출**
`Terminal → New Terminal`에서 `claude`를 입력하면 터미널 기반 Claude 인터페이스가 열린다. 여러 개의 터미널 창을 동시에 열어 병렬 작업이 가능하다.

### 3.3 Agent Manager: 핵심 파워 기능

Agent Manager는 Antigravity에서 가장 강력한 기능이다. 여러 프로젝트 폴더를 넘나들며 단일 에이전트 세션에서 다양한 작업을 처리할 수 있다.

**실전 활용 예시:**
```
Agent Manager에서 Business 폴더를 클릭한 후:
"UFC에게 보낼 $20,000짜리 인보이스를 만들어줘. 
UFC 주소를 온라인에서 찾고, 
내 기존 인보이스 스타일로 데스크탑에 저장해줘."
```

이 한 줄의 명령으로 에이전트는 인보이스 생성부터 저장까지 완료한다. 에이전트는 해당 폴더의 기존 파일들(은행 정보, 주소 등)에 접근하여 작업을 수행한다.

**Inbox 관리:**
모든 에이전트 작업은 Inbox에서 일괄 관리된다. 승인이 필요한 작업들을 한 화면에서 확인하고 연속으로 승인할 수 있어 효율성이 극대화된다.

### 3.4 시각적 인텔리전스(Visualization Intelligence)

코드가 올바르게 동작하는지 확신이 필요할 때 활용하는 고급 기능이다. AI가 생성한 비즈니스 로직을 시각적 플로우차트로 확인하여 의사결정 구조를 직관적으로 검토할 수 있다.

**피자 배달 요금 시스템 예시:**
1. 무료 비주얼 로직 도구에서 요구사항 설명
   - 3마일 초과 시 배달료 2배
   - 프리미엄 회원 20% 할인
   - 한 달 5회 이상 주문 시 추가 할인
2. 도구가 의사결정 플로우차트 생성
3. 각 노드를 클릭해 로직 검증 (노란색 = 조건, 그 외 = 액션)
4. 공유 버튼 → 엔드포인트 복사 → Antigravity에 MCP로 연결

이 방식은 단순한 AI 신뢰("Trust Me Bro") 방식에서 벗어나, **검증 가능한 로직 레이어**를 도입함으로써 기업 환경에서 AI 결과물의 신뢰도를 높이는 핵심 접근법이다.

## 4. 레벨 3: 속도 최적화와 멀티에이전트 활용

### 4.1 병렬 서브에이전트 활용

터미널에서 Claude에게 서브에이전트 기능을 활성화하도록 요청하면 동시에 여러 작업을 병렬 처리할 수 있다.

**예시 프롬프트:**
```
서브에이전트 기능을 활용해서 에이전트 5개를 실행해줘. 
각 에이전트는 서로 다른 세계 수도를 랜덤으로 선택하고, 
그 수도가 왜 최고인지에 대한 500단어 에세이를 작성해서 
왼쪽 파일 목록에 저장해줘.
```

5개의 에이전트가 Tokyo, Reykjavik, Copenhagen, Wellington, Lisbon 등의 수도에 대한 에세이를 동시에 작성한다. 각 에이전트는 독립적인 컨텍스트 윈도우를 가지며, Claude가 오케스트레이터 역할을 수행한다.

### 4.2 1메시지 1목적 원칙

성능 향상을 위한 가장 단순하지만 강력한 규칙이다.

**좋은 예:**
```
"웹사이트에 ABC 기능을 담당하는 새 섹션을 만들어줘."
```

**나쁜 예:**
```
"새 섹션 만들어주고, 컬러 스킴도 바꿔줘."
```

하나의 메시지에 하나의 명확한 목적만 담으면 품질과 속도 모두 향상된다.

### 4.3 루프 탈출 프롬프트

에이전트가 같은 실수를 반복하며 루프에 빠졌을 때 사용하는 특효약 프롬프트다.

```
우리가 루프에 빠진 것 같아. 지금 멈추고 
이 문제를 어떻게 해결할 건지 간결하면서도 상세하게 설명해줘. 
그리고 이번에는 저번과 무엇이 다를 건지도 말해줘.
```

이 프롬프트는 에이전트가 자신의 접근 방식을 메타인지적으로 재평가하도록 강제하여 반복적인 실패 패턴을 끊는다.

## 5. 레벨 4: 디자인 품질 향상 기법

코드 기능 구현에 최적화된 Antigravity와 달리, 시각적 디자인에는 다른 접근이 필요하다.

### 5.1 AI Studio → Antigravity 파이프라인

AI Studio는 Antigravity의 프론트엔드 버전으로, **아름다운 UI 디자인 생성에 특화**되어 있다. 전통적인 의미의 속도 제한이 없으며, 복잡한 애니메이션과 시각적 효과를 훨씬 쉽게 만들 수 있다.

**워크플로우:**
1. AI Studio에서 목표 디자인의 80% 완성
2. "앱 다운로드" 버튼으로 ZIP 파일 저장
3. Antigravity에서 해당 폴더 열기 (`Open Folder`)
4. Antigravity에서 세부 기능 구현 및 최적화

**AI Studio가 Antigravity보다 디자인에 유리한 이유:** AI Studio는 기본적으로 더 많은 시각화 의존성(라이브러리)을 내장하고 있다. Antigravity도 같은 작업이 가능하지만 필요한 패키지를 별도로 설치해야 하는 경우가 많다.

### 5.2 UI 스나이핑(UI Sniping) 전략

마음에 드는 UI 컴포넌트를 다른 사이트에서 "저격"하여 자신의 프로젝트에 통합하는 기법이다.

**주요 소스:**
- **21st.dev**: 큐레이션된 고품질 UI 컴포넌트 모음 (주간 베스트 활용 추천)
- **CodePen.io**: 개발자들이 공유하는 창의적인 HTML/CSS/JS 실험들

**활용 방법:**
1. 원하는 컴포넌트를 찾아 `Share Component` 클릭
2. Antigravity로 이동하여 링크와 함께 "내 웹사이트에 추가해줘" 요청
3. 컴포넌트의 프롬프트까지 함께 복사하면 코드 세부 사항까지 포함

### 5.3 HTML 웹사이트 추출 기법

마음에 드는 웹사이트 구조를 클론하는 방법이다.

1. 마음에 드는 웹사이트에서 URL 복사
2. `HTML Website Extractor` 도구 활용 (Google에서 검색)
3. 소스 코드 다운로드
4. AI Studio에 업로드하여 "이 HTML을 기반으로 더 개선된 버전 만들어줘" 요청

단, 이렇게 추출한 코드는 클라이언트에게 납품하거나 판매해서는 안 된다. 개인 학습 및 UI 참조 목적으로만 활용해야 한다.

### 5.4 Go High Level 및 WordPress 통합

Antigravity에서 만든 웹사이트를 마케팅 플랫폼으로 이전하는 방법이다.

**Go High Level 통합:**
1. Antigravity에서 "이 웹사이트를 HTML과 CSS로 변환해줘" 요청
2. Go High Level에서 `Sites → Websites → Edit`
3. `Add Element → Code` 선택
4. 변환된 HTML/JavaScript 코드 붙여넣기

Go High Level은 HTML과 JavaScript를 네이티브로 지원하므로, Antigravity에서 생성한 코드를 큰 수정 없이 임포트할 수 있다.

### 5.5 Veux Promax 스킬 — 웹사이트 품질 감사

GitHub에서 제공되는 Veux Promax 스킬을 Skills 폴더에 추가하면 50가지 이상의 웹사이트 품질 항목을 자동 점검한다. SEO 최적화, 접근성(Accessibility), 색상 대비, 성능 등 다양한 항목을 자동으로 체크하고 개선 사항을 제안한다.

작업 완료 후 항상 이 프롬프트를 추가하자:
```
"완료 후, 변경한 모든 내용을 녹색 체크 표시와 함께 
체크리스트로 보여줘."
```

### 5.6 스크린샷을 활용한 UI 감사

원하는 UI 수정 사항이 있을 때 텍스트로 설명하는 것보다 훨씬 정확한 방법이다.

- **Mac**: `Command + Shift + 5`로 화면 일부 캡처 후 클립보드에 저장
- **Windows**: `Windows + Shift + S` (Snipping Tool)

캡처한 이미지를 Antigravity 대화창에 붙여넣으면 (`Cmd/Ctrl + V`), 에이전트가 이미지를 직접 분석하여 정확히 어떤 부분을 어떻게 변경해야 하는지 파악한다.

## 6. 레벨 5: 통합과 배포

Antigravity를 외부 세계와 연결하는 전략이다.

### 6.1 MCP 서버 통합의 3단계 접근법

**1단계: 빌트인 MCP 확인**
`⋮ → MCP Servers`에서 먼저 찾는다. Supabase 같은 주요 서비스는 원클릭 설치가 가능하다. 토큰을 생성하고 복사하여 연결하면 끝이다.

**2단계: MCP 마켓플레이스 탐색**
빌트인에 없다면 `mcpmarket.com`에서 찾는다. 인기도 순으로 정렬되어 있으며, 검증된 개발자의 MCP만 설치하는 것이 중요하다. GitHub 리뷰를 반드시 확인하고, 프롬프트 인젝션 위험이 있는 검증되지 않은 MCP는 피해야 한다.

**3단계: 수동 설치**
원하는 MCP의 README에서 설정 코드 복사 → Antigravity에서 `MCP config.json`에 추가 요청.

### 6.2 추천 MCP 서버 목록

Jack Roberts가 실전에서 가장 많이 활용하는 MCP 목록이다.

| MCP | 기능 |
|-----|------|
| **Apify** | 웹 스크래핑 (어떤 웹사이트든 스크래핑 가능) |
| **Context 7** | 최신 API 문서 자동 확인 (Stripe, OpenAI 등 API 변경 즉시 반영) |
| **Supabase** | 데이터베이스 및 스토리지 |
| **Vercel** | 웹사이트 호스팅 및 원클릭 배포 |
| **Notion** | 문서 및 데이터베이스 관리 |
| **Pinecone** | 장기 메모리 (벡터 데이터베이스) |
| **Zapier** | 8,000개 이상의 앱 연결 게이트웨이 |
| **Notebook LM** | 지식 관리 |

**Context 7의 중요성:** AI 모델의 학습 데이터는 특정 시점에 고정되어 있어, Stripe가 내일 API를 변경해도 Antigravity는 모를 수 있다. Context 7은 항상 최신 API 문서를 실시간으로 참조하여 이 문제를 해결한다.

### 6.3 Zapier MCP — 8,000개 앱 통합

Zapier MCP를 연결하면 사실상 모든 SaaS 서비스와 Antigravity를 연결할 수 있다. Gmail, Google Docs, Google Drive, Slack, Notion 등 수천 가지 서비스에 자연어로 접근 가능하다.

**설정 방법:**
1. `mcp.zapier.com/servers` 접속 후 로그인
2. `Add Tool` → 원하는 앱 선택 (예: Gmail)
3. 원하는 액션 선택 (수신, 발송, 초안 작성 등)
4. `Connect` 클릭 → OAuth 인증 완료
5. Antigravity에서 Zapier MCP 활성화

**활용 예시:**
```
Zapier MCP를 사용해서 내 Gmail 받은 편지함에서 
가장 최근 이메일의 제목을 알려줘.
```

**주의:** MCP 도구 총 개수를 50개 이하로 유지하라. 초과 시 컨텍스트 윈도우에 부담이 생겨 성능이 저하된다.

### 6.4 Vercel 원클릭 배포

Vercel MCP를 연결하면 GitHub 푸시와 동시에 자동으로 프로덕션 배포가 이루어진다.

**토큰 발급 경로:** `vercel.com/account/settings/tokens`

연결 후 이 조합을 스킬로 저장해두면 단 하나의 명령으로 GitHub 푸시와 Vercel 배포를 동시에 처리할 수 있다.

### 6.5 Modal — 서버리스 자동화 실행 환경

웹 스크래퍼나 자동화 스크립트를 노트북이 꺼져 있어도 실행하려면 원격 서버가 필요하다. **Modal**이 이 문제를 해결한다.

Modal은 인프라를 직접 관리하지 않아도 코드를 서버리스로 실행할 수 있게 해주는 플랫폼이다. 무료로 시작할 수 있으며, Antigravity와의 통합이 간단하다.

**설정 방법:**
1. modal.com에서 계정 생성 (무료)
2. `Apps → Quick Start Guide` 코드 복사
3. Antigravity에 붙여넣으면 자동으로 Modal 연결 및 코드 생성

**활용 예시:**
```
이 스크래퍼를 매일 오전 6시에 실행하고, 
결과를 정리해서 리포트로 만들어줘.
```

Modal에서 실행 중인 모든 작업의 상태를 대시보드에서 확인할 수 있으며, 실패 시 알림 설정도 가능하다.

## 7. 레벨 6: 비용 절감 전략

성능을 유지하면서 토큰 사용량과 비용을 최소화하는 전략이다.

### 7.1 claude.md / gemini.md 파일 최적화

Antigravity는 모든 메시지에 프로젝트 헌법 파일(`claude.md` 또는 `gemini.md`)을 자동으로 포함시킨다. 이 파일이 700줄이라면 매 메시지마다 700줄이 컨텍스트로 전송된다.

**해결책:** 이 파일을 최대한 간결하게 유지하라. 불필요한 내용을 제거하고 핵심 지침만 남기는 것이 토큰 절약의 첫 번째 방법이다.

### 7.2 대화창 요약 후 새로 시작

긴 대화를 그대로 이어가는 것은 비효율적이다. 다음 프롬프트로 대화 내용을 압축하라:

```
이 대화를 간결하게 요약해줘. 문제점, 진행 상황, 
다음 단계를 포함해서 새로운 에이전트가 
이어받을 수 있을 정도로 간결하게 정리해줘.
```

이 출력물을 복사하고 새 대화창(`+`)을 열어 붙여넣으면 깨끗한 컨텍스트로 작업을 이어갈 수 있다.

### 7.3 작업 유형별 모델 선택 전략

"알버트 아인슈타인이 잔디를 깎을 필요는 없다." 모든 작업에 최고 성능 모델을 사용하는 것은 낭비다.

| 작업 유형 | 추천 모델 | 이유 |
|-----------|-----------|------|
| 초기 프로젝트 계획 | Gemini 3 Deep Think / Claude Opus | 복잡한 추론 필요 |
| 코드 구현 | Gemini 3 Pro (Fast Mode) | 충분한 성능, 빠른 속도 |
| 간단한 리서치 | GPT-OSS | 토큰 비용 절약 |
| 빠른 수정/단순 쿼리 | Fast Mode | 계획 단계 불필요 |

**실전 팁:** Planning Mode는 복잡한 작업의 첫 메시지에서만 사용하고, 이후 반복적인 구현 작업은 Fast Mode로 전환하라.

### 7.4 MCP 도구 수 최적화

Antigravity가 보유한 MCP 도구 목록 전체가 컨텍스트 윈도우에 포함된다. 67개의 활성 도구는 과도하게 많다.

**관리 전략:**
- 현재 사용하지 않는 MCP는 비활성화
- Zapier처럼 수십 가지 서브 도구를 가진 MCP는 필요한 것만 선택적으로 활성화
- **레이지 로더(Lazy Loader)** 활용: 필요할 때만 MCP를 동적으로 로드하여 컨텍스트 절약

### 7.5 Open Code 활용

토큰이 소진될 위기에 처했을 때의 대안 전략이다. 터미널에서 `open code`를 설치하면 수백 가지 프리미엄 모델을 무료 또는 저비용으로 접근할 수 있다. ChatGPT Plus/Pro 구독이 있다면 해당 구독을 코딩에 활용하는 것도 가능하다.
