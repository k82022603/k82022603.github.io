## 1. 이 시스템이 필요한 이유

### AI 도구의 근본적인 한계: 기억 상실

Claude Code, ChatGPT 등 현대의 AI 도구들은 놀랍도록 강력하지만, 공통된 치명적 약점을 가지고 있습니다. 바로 **세션 간 기억이 없다**는 점입니다. 오늘 AI와 나눈 대화, 설정한 컨텍스트, 알려준 작업 스타일은 내일 새 세션을 열면 완전히 사라집니다. 매번 "나는 이런 사람이고, 이런 프로젝트를 하고 있고, 이런 방식으로 일해"를 처음부터 반복해야 합니다.

이 문제를 해결하는 가장 우아한 방법이 바로 **Obsidian Vault를 AI의 외부 장기 기억으로 활용**하는 것입니다.

### 정보 파편화 문제

현대 지식 작업자들은 수십 개의 앱, 수백 개의 탭, 수천 개의 파일 사이에서 일합니다. 아이디어는 카카오톡 메모에, 할 일은 노션에, 참고 논문은 다운로드 폴더에, 코드 스니펫은 GitHub에 흩어져 있습니다. 컨텍스트 전환이 어려운 사람일수록 이 파편화는 생산성에 치명적입니다.

Claude Code + Obsidian 시스템은 이 모든 것을 **단 하나의 마크다운 기반 볼트(Vault)** 로 통합합니다.

### ahrom.insights의 증언

> "내가 ADHD인이라서 컨텍스트 스위치를 잘 못해. 시도때도 없이 생각이 여기저기로 튀는데 이 모든걸 한곳에서 볼 시스템이 항상 필요했거든. 근데 옵시디언+클코가 그걸 해냄... 것도 3일만에."

이 시스템의 핵심 가치는 **속도**와 **통합**입니다. 생각이 떠오르는 순간, 명령어 하나로 적절한 위치에 자동 분류·저장됩니다.

## 2. 시스템 구조 개요

이 워크플로우는 세 개의 층으로 이루어져 있습니다.

### 층 1: 입력 계층 (Claude Code Inputs)

Claude Code는 터미널에서 실행되는 AI 코딩·작업 에이전트입니다. 다음과 같은 다양한 입력을 처리합니다.

- URL 및 웹 링크
- 메모와 아이디어 텍스트
- PDF 파일
- 논문 및 연구 자료
- 할 일 목록과 프로젝트 정보

### 층 2: 처리 계층 (Claude Code + 슬래시 명령어)

터미널에서 실행하는 커스텀 슬래시 명령어들이 입력을 자동으로 분류·처리·저장합니다. 이 명령어들은 CLAUDE.md 파일에 정의되어 있으며, AI가 컨텍스트를 유지하면서 일관된 방식으로 동작합니다.

### 층 3: 저장 계층 (Obsidian Vault & Knowledge Base)

처리된 모든 결과물은 Obsidian Vault의 마크다운 파일로 저장됩니다. Obsidian은 이 파일들 사이의 연결 관계를 시각화하고, 검색하고, 그래프 뷰로 탐색할 수 있게 해줍니다.

### 핵심 명령어 요약표

| 명령어 | 주요 기능 | 출력 위치 |
|--------|-----------|-----------|
| `/today` | 아침 브리핑 및 할 일 생성 | `13-Tasks/daily/` |
| `/capture` | URL 및 메모를 지식 베이스에 저장 | `10-Knowledge/` |
| `/weekly-digest` | 한 주 요약 + 다음 주 미리보기 | `13-Tasks/weekly/` |
| `/research` | 키워드 기반 논문 자동 수집·요약 | `10-Knowledge/` |
| `/process-inbox` | 받은 함 항목 분류 및 정리 | 각 주제 폴더 |
| `/learned` | 배운 내용 컨텍스트 파일에 저장 | `10-Knowledge/` |
| `/build-log` | 작업 내역 자동 문서화 | `build.log/` |

## 3. 일간 워크플로우 (Daily Workflow)

하루는 크게 세 개의 루틴으로 구성됩니다: 아침 브리핑, 작업 중 캡처, 저녁 마무리.

---

### ☀️ 아침 루틴 (5분): 브리핑 및 집중

하루를 시작할 때 터미널을 열고 단 하나의 명령어를 실행합니다.

```bash
/today
```

이 명령어는 다음을 자동으로 수행합니다.

첫째, 오늘 날짜 기반의 일일 파일을 생성합니다. 파일명은 `13-Tasks/daily/YYYY-MM-DD-briefing.md` 형식으로 자동 생성됩니다.

둘째, Obsidian Vault 내에 저장된 이번 주 목표와 지난주 작업 내역을 참조하여 오늘 우선 처리해야 할 항목을 제안합니다.

셋째, `/research` 명령어를 통해 사전에 등록된 관심 키워드(예: "AI Agent", "ADHD 생산성", "RAG 시스템" 등 30개)로 arXiv 등에서 수집된 논문 요약을 포함합니다.

결과물은 다음과 같은 형태의 마크다운 파일입니다.

```markdown
# 2025-03-18 브리핑

## 📌 오늘의 할 일
- [ ] 할 일 1
- [ ] 할 일 2
- [ ] 할 일 3

## 📚 오늘의 논문 요약
- [논문 제목] - 핵심 인사이트 요약

## 🎯 이번 주 목표 진행 현황
- 목표 1: 70% 완료
```

이 5분 루틴의 핵심 가치는 매일 아침 새로 "뭘 해야 하지?"를 고민하는 시간을 제거하는 것입니다. AI가 어제의 작업 맥락을 기억하고 오늘의 집중점을 자동으로 제시해줍니다.

---

### 📎 작업 중 루틴: 실시간 캡처 및 생각 정리

업무 중 좋은 아이디어, 참고할 URL, 처리해야 할 생각이 떠오르면 즉시 캡처합니다.

#### `/capture` 명령어

```bash
/capture https://arxiv.org/abs/2312.XXXXX "RAG 성능 개선에 활용할 수 있는 논문"
```

이 명령어는 다음을 수행합니다.

URL의 경우 해당 페이지를 읽고 핵심 내용을 요약한 후, 볼트 내 20개의 주제 폴더 중 가장 적합한 곳에 자동 분류하여 마크다운 파일로 저장합니다. 단순 메모의 경우도 동일하게 주제를 파악하여 분류합니다.

예를 들어 AI 관련 자료는 `10-Knowledge/AI-Research/`, 생산성 관련 자료는 `10-Knowledge/Productivity/`에 저장되는 식입니다.

#### `/process-inbox` 명령어

하루 동안 빠르게 메모해 둔 미처리 항목들을 한 번에 정리할 때 사용합니다.

```bash
/process-inbox
```

`process-inbox/thought.md` 파일에 쌓인 모든 항목을 AI가 읽고, 각각을 적절한 폴더로 분류하여 이동시킵니다. "나중에 정리해야지"라는 생각을 실제로 자동화한 것입니다.

---

### 🌙 저녁 루틴 (5분): 학습 내용 및 작업 로그 기록

하루를 마무리할 때 두 개의 명령어를 실행합니다.

#### `/learned` 명령어

오늘 대화하면서, 작업하면서 새로 알게 된 내용을 지식 베이스에 저장합니다.

```bash
/learned "오늘 Claude Code의 CLAUDE.md가 세션 시작 시 자동으로 컨텍스트에 로드된다는 것을 알게 됨"
```

이 내용은 `10-Knowledge/learning-log.md`에 날짜와 함께 누적 저장됩니다. 시간이 지날수록 나만의 지식 데이터베이스가 쌓입니다.

#### `/build-log` 명령어

오늘 만든 것, 완성한 것, 진행한 프로젝트를 자동으로 문서화합니다.

```bash
/build-log
```

현재 세션의 작업 내역을 분석하여 `build.log/YYYY-MM-DD.md` 파일에 기록합니다. "오늘 뭘 했는지" 기억하지 못하는 문제를 해결합니다.

## 4. 주간 워크플로우 (Weekly Workflow)

### 월~금: 요일별 집중 루틴

각 요일마다 특정 작업에 집중하는 리듬을 만듭니다.

**월요일 - 한 주의 방향 설정 및 우선순위 확정**

`/today` 명령어로 이월 작업을 확인하고, 이번 주의 핵심 목표 2~3개를 선정합니다. 월요일에 명확한 목표를 설정하면 나머지 요일의 결정 피로도가 크게 줄어듭니다.

**수요일 - 중간 점검 및 정보 정리**

`/process-inbox`로 주 중반까지 누적된 항목을 정리하고, 진행이 뒤처지는 업무가 없는지 확인합니다. 이 단계에서 필요하면 우선순위를 재조정합니다.

**금요일 - 한 주 요약 및 성과 기록**

가장 중요한 주간 루틴입니다. `/weekly-digest` 명령어를 실행하여 한 주를 마무리합니다.

```bash
/weekly-digest
```

이 명령어는 다음을 자동으로 수행합니다.

이번 주에 완료한 작업 목록을 취합하고, 수집된 논문 및 자료를 요약하며, `13-Tasks/weekly/weekly-summary.md` 파일에 다음 주 미리보기를 포함한 종합 리포트를 생성합니다. 또한 `/learned`로 기록된 학습 내용을 취합하여 이번 주에 성장한 부분을 시각화합니다.

### 주말: 생활 관리 및 시스템 최적화

**일요일 - 다음 주 프리뷰 및 생활 관리**

다음 주 일정을 미리 살피고, 냉장고 매니저 도구를 통해 식단 및 장보기 계획을 세웁니다. 이는 업무뿐 아니라 일상 생활까지 시스템화한다는 이 워크플로우의 핵심 철학을 보여줍니다.

**매월 1일 - 재무 및 프로젝트 전면 검토**

'돈 지킴이' 도구로 예산 대비 지출을 분석하고, 정제된 프로젝트의 보관 여부를 결정합니다. 월간 단위로 시스템 자체를 돌아보고 개선합니다.

### Librarian 시스템 유지보수

일주일간 새로 추가된 PDF 자료를 색인화하여 지식 베이스의 최신성을 유지합니다. 이를 통해 지식 베이스가 계속 살아있는 자료로 유지됩니다.

## 5. 핵심 슬래시 명령어 7가지 상세 설명

슬래시 명령어는 Claude Code의 커스텀 명령어 시스템을 활용하여 만들어집니다. 각 명령어는 `.claude/commands/` 디렉토리에 마크다운 파일로 정의됩니다.

---

### `/today` - 아침 브리핑 생성기

**역할:** 오늘 할 일 + 이번 주 목표 + 논문 요약을 자동 생성합니다.

**동작 방식:** 명령어가 실행되면 AI는 Vault 내 지난 며칠간의 파일을 읽어 미완료 항목을 파악하고, 이번 주 목표 파일을 참조하며, 키워드 기반으로 수집된 논문 요약을 포함한 종합 브리핑 파일을 생성합니다.

**출력 위치:** `13-Tasks/daily/YYYY-MM-DD-briefing.md`

**실용 팁:** 이 명령어를 아침에 커피를 내리는 동안 실행해두면, 자리에 앉았을 때 이미 오늘의 집중점이 화면에 펼쳐져 있습니다.

---

### `/capture` - 즉시 캡처 및 자동 분류

**역할:** URL이나 메모를 던지면 20개 주제 폴더에 자동 분류하여 저장합니다.

**동작 방식:** 입력된 URL을 실제로 방문하거나, 텍스트를 분석하여 주제를 파악합니다. 그런 다음 볼트 내 사전 정의된 지식 카테고리(AI, 생산성, 재무, 건강 등)와 매칭하여 가장 적합한 폴더에 요약 마크다운 파일을 생성합니다.

**출력 위치:** `10-Knowledge/[해당 카테고리]/`

**활용 예시:**
```bash
/capture https://example.com/great-article "나중에 블로그 글 쓸 때 참고"
/capture "RAG 시스템에서 청크 사이즈는 256~512 토큰이 최적이라는 연구 결과"
```

---

### `/research` - 논문 자동 수집기

**역할:** 사전에 등록된 키워드 30개로 arXiv 등에서 논문을 자동 수집하고 요약합니다.

**동작 방식:** CLAUDE.md 또는 별도 설정 파일에 관심 키워드를 등록해두면, 이 명령어 실행 시 해당 키워드로 학술 데이터베이스를 검색하고 관련 논문을 수집하여 핵심 내용을 요약한 마크다운 파일을 생성합니다.

**출력 위치:** `10-Knowledge/Research/`

**설정 방법:** `CLAUDE.md` 또는 별도 `research-keywords.md` 파일에 키워드 목록을 관리합니다.

---

### `/process-inbox` - 인박스 일괄 처리

**역할:** 쌓여있는 인박스 항목을 AI가 분류하고 정리합니다.

**동작 방식:** `process-inbox/` 폴더나 지정된 인박스 파일에 쌓인 미처리 메모, URL, 생각들을 AI가 일괄 처리합니다. 각 항목의 성격을 파악하여 해당하는 지식 폴더, 할 일 목록, 또는 프로젝트 파일로 이동시킵니다.

**활용 팁:** 바쁠 때는 그냥 `/capture`로 던져두고, 여유 있을 때 `/process-inbox`로 한 번에 정리합니다. GTD(Getting Things Done) 방법론의 "수집 → 처리" 단계를 자동화한 것입니다.

---

### `/learned` - 학습 내용 자동 저장

**역할:** 대화에서 배운 것, 작업 중 깨달은 것을 컨텍스트 파일에 자동 저장합니다.

**동작 방식:** 입력된 학습 내용을 날짜와 함께 `10-Knowledge/learning-log.md`에 추가합니다. 시간이 지날수록 개인화된 지식 데이터베이스가 형성됩니다.

**출력 위치:** `10-Knowledge/learning-log.md`

**중요성:** 이 명령어는 단순 저장을 넘어, 다음 세션에서 AI가 CLAUDE.md를 통해 이 학습 내용을 참조할 수 있게 만듭니다. AI가 "내가 이미 아는 것"을 인식하게 되는 메커니즘입니다.

---

### `/build-log` - 작업 내역 자동 문서화

**역할:** 오늘 작업한 내용을 자동으로 문서화합니다.

**동작 방식:** 현재 세션의 작업 내역을 분석하여 "무엇을 만들었는지", "왜 만들었는지", "어떤 결정을 내렸는지"를 정리한 로그 파일을 생성합니다. 개발 일지이자 포트폴리오의 원재료가 됩니다.

**출력 위치:** `build.log/YYYY-MM-DD.md`

---

### `/weekly-digest` - 주간 종합 리포트

**역할:** 한 주 요약 + 다음 주 미리보기를 생성합니다.

**동작 방식:** 한 주간 생성된 모든 일일 파일, 캡처 자료, 학습 내용, 빌드 로그를 종합하여 주간 리포트를 생성합니다. 완료한 일, 배운 것, 다음 주 계획이 한 페이지에 정리됩니다.

**출력 위치:** `13-Tasks/weekly/weekly-summary.md`

**금요일 활용:** 매 금요일 퇴근 전 이 명령어를 실행하는 것을 습관으로 만들면, 주 단위 회고와 계획이 자동화됩니다.

## 6. 에이전트 시스템: PDF Sorter & Librarian

이 시스템의 숨겨진 보석은 PDF 자동 분류 에이전트입니다.

### 작동 방식

구글 드라이브나 원드라이브의 지정된 폴더에 PDF를 넣으면, AI 에이전트가 자동으로 다음을 수행합니다.

**1단계 - 읽기:** PDF의 내용을 전체 분석합니다. 단순 OCR을 넘어 논문의 경우 제목, 저자, 초록, 핵심 주장, 방법론, 결론을 파악합니다.

**2단계 - 분류:** 내용을 바탕으로 볼트 내 적절한 카테고리를 결정합니다. AI, 생산성, 의료, 경제, 철학 등 사전에 정의된 분류 체계에 따라 배치됩니다.

**3단계 - 태그 달기:** 검색을 용이하게 하는 메타데이터 태그를 자동 부여합니다.

**4단계 - 요약 생성:** Obsidian에 해당 PDF의 요약 `.md` 파일을 생성합니다. 이 파일은 PDF의 핵심 내용을 빠르게 파악할 수 있게 해주며, 다른 노트와 자동으로 연결됩니다.

### 실제 효과

ahrom.insights 님의 경우, 이 시스템으로 **PDF 400개를 자동 분류**했습니다. 다운로드 폴더와 데스크탑에 수백 개씩 흩어져 있던 파일들이 체계적인 지식 베이스로 변환된 것입니다.

### 일주일 유지보수

매주 Librarian 에이전트가 새로 추가된 PDF를 색인화하여 지식 베이스의 최신성을 유지합니다. 사용자는 PDF를 지정 폴더에 드롭하기만 하면 됩니다.

이 도구들은 Claude Code의 슬래시 명령어나 에이전트 스크립트로 구현되며, Obsidian Vault와 연동되어 모든 데이터가 한 곳에서 관리됩니다.

## 9. CLAUDE.md: AI의 장기 기억 설정

`CLAUDE.md`는 이 시스템에서 가장 중요한 파일입니다. Claude Code는 세션 시작 시 이 파일을 자동으로 컨텍스트에 로드합니다. 즉, 이 파일에 적힌 내용은 AI가 항상 "기억"하는 정보가 됩니다.

### CLAUDE.md 구성 요소

효과적인 CLAUDE.md는 다음 섹션으로 구성됩니다.

**나에 대한 정보:** 이름, 직업, 작업 스타일, ADHD나 특정 조건 등 AI가 알아야 할 기본 정보를 기록합니다.

**현재 프로젝트:** 진행 중인 프로젝트 목록과 각 프로젝트의 핵심 컨텍스트를 기록합니다. AI가 항상 현재 상황을 파악하고 있게 됩니다.

**Vault 구조 설명:** 어떤 폴더에 어떤 자료가 있는지 설명합니다. AI가 파일을 정확한 위치에 저장할 수 있게 됩니다.

**명령어 정의:** 각 슬래시 명령어가 무엇을 하는지, 어떤 형식으로 출력하는지 정의합니다.

**작업 선호도:** 코드 스타일, 문서 형식, 언어 선호도 등 AI의 출력 방식을 커스터마이즈합니다.

**연구 키워드:** `/research` 명령어에서 사용할 관심 분야 키워드 목록을 관리합니다.

### CLAUDE.md 예시 구조

```markdown
# 나의 Claude Code 컨텍스트

## 나에 대해
- 이름: [이름]
- 직업: AI 연구자 / 콘텐츠 크리에이터
- 특성: ADHD로 인해 컨텍스트 전환이 어려움. 단일 시스템 선호.
- 선호 언어: 한국어 (영어 자료 한국어 요약)

## 현재 진행 중인 프로젝트
1. [프로젝트명]: [핵심 설명]
2. [프로젝트명]: [핵심 설명]

## Vault 구조
- 10-Knowledge: 지식 베이스 (20개 카테고리)
- 13-Tasks/daily: 일일 브리핑
- 13-Tasks/weekly: 주간 리포트
- build.log: 작업 내역

## 슬래시 명령어
[각 명령어 정의]

## 연구 관심 키워드
AI Agent, RAG, ADHD, 생산성, [기타 키워드들]
```

## 10. 실전 설정 방법 (단계별 가이드)

### 사전 준비물

이 시스템을 시작하기 전에 다음이 필요합니다.

- **Claude Code:** Anthropic의 터미널 기반 AI 도구. npm을 통해 설치하거나 Anthropic 공식 사이트에서 다운로드합니다.
- **Obsidian:** 무료 마크다운 노트 앱. obsidian.md에서 다운로드합니다.
- **Node.js:** Claude Code 실행을 위해 필요합니다.
- **Claude API 키 또는 Claude Max 구독:** Claude Code 사용을 위해 필요합니다.

### 1단계: Obsidian Vault 생성

Obsidian을 설치하고 새 Vault를 생성합니다. 위에서 제시한 폴더 구조를 참고하여 기본 폴더 구조를 만듭니다. 이 폴더들은 Obsidian 내에서 직접 만들거나, 터미널에서 `mkdir` 명령어로 일괄 생성할 수 있습니다.

### 2단계: Claude Code 설치 및 초기화

터미널을 열고 Vault 디렉토리로 이동한 후 Claude Code를 실행합니다.

```bash
cd ~/path/to/your/vault
claude
```

처음 실행 시 `/init` 명령어를 실행하면 기본 CLAUDE.md 파일과 `.claude/` 폴더 구조가 생성됩니다.

```bash
/init
```

### 3단계: CLAUDE.md 커스터마이즈

생성된 CLAUDE.md 파일을 열어 본인의 정보, 프로젝트, 선호도를 입력합니다. 이 파일이 구체적이고 상세할수록 AI의 도움이 더 정확해집니다.

### 4단계: 슬래시 명령어 정의

`.claude/commands/` 폴더에 각 명령어를 정의하는 마크다운 파일을 생성합니다. 예를 들어 `today.md` 파일에는 `/today` 명령어 실행 시 AI가 수행할 작업을 상세히 기술합니다.

```markdown
# /today 명령어

이 명령어가 실행되면 다음을 수행하세요:

1. 오늘 날짜의 브리핑 파일을 `13-Tasks/daily/YYYY-MM-DD-briefing.md`에 생성
2. `13-Tasks/weekly/` 폴더에서 이번 주 목표를 읽어 참조
3. `10-Knowledge/Research/` 폴더에서 최신 논문 요약을 포함
4. 미완료 할 일 목록 포함
5. 아래 형식으로 출력:
   [형식 정의]
```

### 5단계: MCP 연동 (선택 사항)

더 강력한 통합을 원한다면 Obsidian MCP 플러그인을 설치합니다. `obsidian-claude-code-mcp` 플러그인은 Claude Code가 WebSocket을 통해 Vault에 직접 접근할 수 있게 해줍니다. 이를 통해 Claude Code가 Vault를 작업 디렉토리로 열지 않아도 노트를 읽고 쓸 수 있습니다.

### 6단계: 일주일 시범 운영

처음 일주일은 시스템을 그대로 사용하면서 불편한 점을 발견합니다. CLAUDE.md를 조금씩 수정하고, 명령어 정의를 개선하면서 자신에게 맞는 시스템으로 발전시킵니다. ahrom.insights 님도 "이 시스템은 아직도 업데이트 중"이라고 밝혔듯, 완성형이 아닌 계속 진화하는 시스템입니다.

## 11. ADHD 친화적 시스템 설계 원칙

이 워크플로우는 특히 ADHD를 가진 사용자를 위해 설계된 특성을 가지고 있습니다. 하지만 이 원칙들은 ADHD 여부와 무관하게 모든 지식 작업자에게 유효합니다.

### 원칙 1: 마찰(Friction) 최소화

좋은 아이디어가 떠올랐을 때, 이를 기록하는 과정이 복잡하면 기록을 포기하게 됩니다. 이 시스템은 명령어 하나(`/capture`)로 즉시 처리합니다. "나중에 정리해야지"라는 생각이 들 틈이 없습니다.

### 원칙 2: 단일 진실 공급원 (Single Source of Truth)

정보가 여러 곳에 흩어져 있으면 "어디에 저장했더라"라는 상황이 발생합니다. 이 시스템은 모든 것을 Obsidian Vault 하나로 통합합니다.

### 원칙 3: 자동화된 구조

빈 파일을 열고 무엇을 써야 할지 고민하는 것은 ADHD에게 특히 어렵습니다. 이 시스템은 구조화된 템플릿을 자동으로 생성합니다.

### 원칙 4: 컨텍스트 유지

세션이 바뀌어도 CLAUDE.md를 통해 AI가 나의 상황을 기억합니다. "지난번에 말했잖아"를 반복하지 않아도 됩니다.

### 원칙 5: 점진적 공개 (Progressive Disclosure)

아침에는 오늘 할 일만, 저녁에는 학습 내용만 보입니다. 모든 것을 한꺼번에 보여주지 않아 인지 과부하를 방지합니다.

## 12. 자주 묻는 질문 (FAQ)

**Q: Claude Code와 Claude 데스크탑 앱의 차이는 무엇인가요?**

Claude 데스크탑 앱은 GUI 기반으로 직관적이지만, 파일 시스템에 대한 접근이 제한적입니다. Claude Code는 터미널 기반으로 파일 읽기/쓰기, 명령어 실행, 스크립트 운영 등 훨씬 강력한 자동화가 가능합니다. ahrom.insights 님도 처음에는 데스크탑 앱으로 시작했다가 터미널 Claude Code로 전환했습니다.

**Q: 프로그래밍 지식이 없어도 이 시스템을 구축할 수 있나요?**

기본 시스템 구축에는 코딩 지식이 크게 필요하지 않습니다. CLAUDE.md와 슬래시 명령어 정의 파일은 자연어로 작성하면 됩니다. 다만 PDF Sorter처럼 외부 서비스와 연동하는 고급 기능은 일부 기술적 설정이 필요할 수 있습니다.

**Q: 데이터 보안은 어떻게 되나요?**

Obsidian은 기본적으로 모든 데이터를 로컬에 저장합니다. 클라우드 동기화를 사용하지 않으면 인터넷에 데이터가 올라가지 않습니다. 단, Claude Code 사용 시 Anthropic 서버와 통신이 이루어지므로, 민감한 개인 정보는 별도 처리가 필요합니다.

**Q: Obsidian 동기화는 어떻게 하나요?**

Obsidian Sync(유료)를 사용하거나, iCloud, Google Drive, Dropbox 등을 통해 볼트 폴더를 동기화할 수 있습니다. macOS와 iOS 기기 간에는 iCloud 동기화가 가장 안정적입니다.

**Q: 이 시스템을 팀에서도 사용할 수 있나요?**

가능합니다. 공유 Obsidian 볼트를 Git이나 클라우드 스토리지로 동기화하면 팀원들이 동일한 지식 베이스를 공유할 수 있습니다. 다만 개인화된 CLAUDE.md는 각자 관리하는 것이 좋습니다.

## 별첨: Obsidian이란 무엇인가?

### 한 줄 정의

Obsidian은 **마크다운 기반의 로컬 노트 앱**으로, 노트 간의 연결 관계를 시각화하는 "제2의 뇌(Second Brain)" 구축 도구입니다.

### 핵심 특징

**마크다운 기반:** 모든 노트는 일반 `.md` 파일로 저장됩니다. 특정 앱에 종속되지 않으며, 어떤 텍스트 에디터로도 열 수 있습니다. 이것이 Claude Code와의 연동이 자연스러운 이유입니다. AI는 이미 마크다운을 완벽하게 이해하기 때문입니다.

**로컬 저장:** 클라우드가 아닌 사용자의 컴퓨터에 직접 저장됩니다. 인터넷 연결 없이도 사용 가능하며, 데이터 주권이 사용자에게 있습니다.

**양방향 링크(Backlinks):** `[[노트 이름]]` 형식으로 노트 간에 링크를 만들 수 있습니다. A 노트에서 B 노트를 링크하면, B 노트에서 "A 노트에서 나를 링크했다"는 역방향 링크도 자동으로 생성됩니다.

**그래프 뷰:** 모든 노트와 링크의 연결 관계를 네트워크 그래프로 시각화합니다. 지식이 어떻게 연결되어 있는지 한눈에 볼 수 있습니다.

**Vault 시스템:** 특정 폴더를 "Vault"로 지정하면, 그 안의 모든 마크다운 파일을 Obsidian이 관리합니다. Claude Code는 이 Vault 폴더를 파일시스템으로 직접 읽고 쓸 수 있습니다.

**플러그인 생태계:** 커뮤니티 플러그인을 통해 기능을 무한히 확장할 수 있습니다. 캘린더, 칸반 보드, 데이터베이스, AI 연동 등 수천 개의 플러그인이 존재합니다.

### Obsidian vs 다른 노트 앱 비교

| 특성 | Obsidian | Notion | Evernote |
|------|---------|--------|---------|
| 데이터 저장 | 로컬 파일 | 클라우드 | 클라우드 |
| 파일 형식 | 마크다운(.md) | 독자 형식 | 독자 형식 |
| AI 연동 | ✅ 매우 용이 | ⚠️ API 필요 | ⚠️ 제한적 |
| 오프라인 사용 | ✅ 완전 지원 | ⚠️ 제한적 | ⚠️ 제한적 |
| 기본 가격 | 무료 | 무료/유료 | 무료/유료 |
| 그래프 뷰 | ✅ 내장 | ❌ 없음 | ❌ 없음 |
| 플러그인 | ✅ 오픈소스 커뮤니티 | ❌ 없음 | ❌ 없음 |

### Claude Code와 Obsidian이 잘 맞는 이유

Obsidian의 볼트 구조는 본질적으로 마크다운 파일로 이루어진 코드베이스와 유사합니다. 파일들이 파일시스템에 저장되어 다른 앱에서도 접근 가능하며, Claude Code는 이미 파일 구조를 탐색하고 정밀한 편집을 수행하는 데 탁월합니다.

또한 Obsidian의 마크다운 기반 볼트 시스템은 AI의 메모리 한계를 극복하는 구조화된 영구 지식 베이스를 만들 수 있게 해주며, 이를 통해 세션 간에도 컨텍스트가 유지됩니다.

### Obsidian 시작하기

1. [obsidian.md](https://obsidian.md) 에서 무료로 다운로드합니다.
2. 새 Vault를 생성합니다(기존 폴더를 지정하거나 새 폴더를 만듭니다).
3. 마크다운 문법을 간단히 학습합니다(`#`은 제목, `**굵은 글씨**`, `[[링크]]` 등).
4. 일단 쓰기 시작합니다. 완벽한 구조는 나중에 만들어집니다.

Obsidian의 철학은 "도구가 사용자의 방식에 맞추는 것"입니다. 처음부터 완벽한 구조를 만들려 하지 말고, 사용하면서 자연스럽게 발전시키는 것이 좋습니다.
