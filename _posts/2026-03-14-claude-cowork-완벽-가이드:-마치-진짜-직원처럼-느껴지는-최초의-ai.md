## 1. 들어가며

AI 도구와 기능은 너무나 빠르게 쏟아져 나오고 있어 따라가기조차 벅차다. 그러나 가끔씩, 실제로 업무 방식 자체를 바꿔버리는 무언가가 등장한다. **Claude Cowork**가 바로 그런 존재다.

Cowork는 단순한 챗봇이 아니다. 이름 그대로 '공동 작업자(co-worker)'에 가깝다. 질문에 답하는 보조 도구가 아니라, 실제로 일을 완수하는 동료처럼 작동한다. 이 문서는 Claude Cowork의 개념, 핵심 기능, 실제 활용 예시, 그리고 현재의 한계까지를 상세하게 다룬다.

## 2. Claude Cowork란 무엇인가?

Claude Cowork는 **사용자의 컴퓨터에 상주하는 AI 에이전트**다. 단순히 텍스트를 생성하거나 질문에 답하는 수준을 넘어, 여러 단계로 구성된 복잡한 작업을 직접 계획하고 실행할 수 있다. 파일 시스템에 접근해 문서를 읽고 쓰며, 캘린더·이메일·드라이브 같은 일상적인 앱들과 직접 연동된다. 사용자가 원하는 결과를 설명하면, Cowork가 스스로 방법을 찾아 실행한다.

### 사용 조건

- **Claude 데스크톱 앱** 설치 필요 (공식 사이트에서 다운로드)
- **유료 플랜 필요**: Pro 또는 Max 플랜 가입자만 사용 가능 (무료 플랜 미지원)

### 인터페이스 구성

Claude 데스크톱 앱을 열면 상단에 세 개의 탭이 있다.

- **Chat 탭**: 기존에 알고 있는 Claude 인터페이스. 일반 대화, 프로젝트 관리 등에 사용
- **Cowork 탭**: 이 가이드의 핵심. 자율적 작업 실행을 위한 에이전트 인터페이스
- **Code 탭**: 코딩을 전혀 몰라도 "이것 만들어줘"라고 말하면 실제로 코드를 작성하고 실행

## 3. Claude Cowork의 핵심 기능

### 3-1. 직접 파일 접근 (Direct File Access)

Cowork는 사용자 컴퓨터의 파일에 직접 접근할 수 있다. 파일을 업로드하거나 다운로드할 필요 없이, 지정된 폴더 안에서 파일을 읽고 수정하고 생성한다. 복잡한 작업을 처리할 때는 여러 하위 에이전트(sub-agents)가 병렬로 동시 작동하여 여러 작업 흐름을 동시에 조율한다.

### 3-2. 전문적인 결과물 생성

포맷이 잘 갖춰진 Word 문서, 스프레드시트, 완전히 편집 가능한 PowerPoint 프레젠테이션 등 실제 업무에서 바로 사용할 수 있는 수준의 결과물을 만들어낸다.

### 3-3. 장시간 작업 처리

타임아웃 없이 장시간이 걸리는 복잡한 작업도 처리할 수 있다. 사용자는 작업을 지시하고 자리를 떠나도 된다. 돌아왔을 때 작업이 완료되어 있다.

### 3-4. 병렬 처리 아키텍처

Claude Code와 동일한 강력한 에이전틱(agentic) 아키텍처를 사용하되, 전문가가 아닌 일반 사용자도 쉽게 쓸 수 있도록 설계되었다. 대규모 작업에서는 파일을 그룹으로 나누어 여러 에이전트가 각자의 배치를 분석한 뒤 결과를 합쳐 한 번에 실행하는 방식으로 효율성을 극대화한다.

## 4. 주요 구성 요소 이해하기

Claude Cowork를 제대로 활용하려면 네 가지 핵심 요소를 이해해야 한다.

### 4-1. 스킬 (Skills)

스킬은 일종의 **레시피 카드**다. Claude가 특정 작업을 반복적이고 일관되게 수행할 수 있도록 전문화된 지침을 담고 있다. 예를 들어 PowerPoint를 만드는 스킬, 특정 브랜드 보이스로 글을 쓰는 스킬, 데이터를 특정 방식으로 분석하는 스킬 등이 있다. Claude는 작업에 관련된 스킬을 자동으로 판단하여 불러오므로 사용자가 따로 신경 쓸 필요가 없다. 물론 슬래시(/) 명령어로 특정 스킬을 수동으로 지정하는 것도 가능하다.

스킬에는 두 가지 유형이 있다.

- **내장 스킬(Built-in Skills)**: Anthropic이 직접 관리하며 모든 사용자에게 자동으로 제공
- **커스텀 스킬(Custom Skills)**: 사용자 또는 조직이 자신의 특정 워크플로우에 맞게 직접 만들어 사용

### 4-2. 플러그인 (Plugins)

플러그인은 여러 스킬을 하나로 묶은 **통합 패키지**다. 단일 레시피 카드가 아니라, 특정 직무 전체를 위해 구성된 요리책에 비유할 수 있다. 영업, 법무, 재무, 마케팅 등 각 분야에 특화된 플러그인을 설치하면 Claude가 해당 워크플로우 전체에 즉시 최적화된다. GitHub 등에서 다른 사람들이 만들어 공유한 플러그인을 설치할 수도 있다. 단, 신뢰할 수 있는 출처에서만 가져와야 한다.

### 4-3. 커넥터 (Connectors)

커넥터는 Claude가 외부 앱과 서비스에 연결되는 방식이다. 주요 커넥터로는 Gmail, Google Drive, Google Calendar 등이 있으며, 그 외에도 Slack, Notion, Gamma, Granola 등 매우 다양한 서비스를 지원한다. 커넥터 덕분에 Cowork는 기존의 업무 흐름 안에 자연스럽게 녹아든다. 별도의 도구로 컨텍스트 전환을 해야 하는 것이 아니라, 이미 쓰고 있는 앱들 속에서 함께 작동한다.

### 4-4. Claude in Chrome (클로드 인 크롬)

웹 브라우징 기능을 Cowork에 제공하는 **크롬 확장 프로그램**이다. 설치 후 토글을 켜두기만 하면 Cowork가 필요할 때 자동으로 웹을 탐색하고 정보를 수집한다.

## 5. 실제 활용 사례 1: 스크린샷 자동 정리

### 문제 상황

사용자는 썸네일 영감을 위해 수많은 이미지를 저장해두었다. YouTube 홈 화면 스크린샷, 뉴스레터 캡처, 트위터 게시물, 광고 이미지 등 345개의 파일이 아무런 분류 없이 한 폴더에 뒤섞여 있었다.

### 작업 지시 방법

1. 대상 폴더를 먼저 복제(중요: 원본 보호)한 뒤 Cowork에 폴더를 지정
2. 폴더 안의 내용물과 원하는 정리 방식을 자연어로 설명

> *"이 폴더 안에는 썸네일 영감을 위한 많은 스크린샷과 이미지가 있습니다. 대부분은 기존 썸네일이나 광고입니다. YouTube 홈페이지, 뉴스레터, 트위터 등에서 가져온 것들입니다. 완전히 정리되어 있지 않습니다. 무엇인지 한눈에 알 수 있도록 파일명을 변경하고, 나중에 영감을 얻기 위해 탐색하기 쉬운 폴더 구조로 정리해주세요."*

### 처리 과정

Cowork는 파일 접근 허가를 요청한 뒤, 폴더 구성 방식(스타일 및 유형별)에 대해 한 번 확인 질문을 했다. 이후 스스로 작업을 설계했다.

- 345개 파일을 여러 배치로 나눔
- 여러 에이전트가 병렬로 각 배치 분석
- 분석 결과를 통합하여 폴더 구조 생성
- 모든 파일을 적절히 이름 변경 및 이동
- 최종 검증 완료

### 결과

- **12개의 스타일별 하위 폴더** 자동 생성
- **345개 파일 전체** 내용을 한눈에 알 수 있도록 이름 변경
- 어떤 파일인지 설명하는 요약 표 제공
- 단 하나의 프롬프트, 최소한의 지시만으로 완전히 자동화

## 6. 작업 예약 기능 (Scheduled Tasks)

정리 작업이 완료된 후, 사용자는 같은 폴더를 매월 자동으로 정리하도록 예약할 수 있었다.

> *"매월 1일에 이 폴더를 확인하고, 내가 새로 추가한 이미지들을 분류하여 이름을 변경하고 적절한 폴더로 이동해주세요."*

이 한 문장으로 **매월 1일 오전 9시에 자동 실행**되는 정기 작업이 설정되었다. 이제 사용자는 마음에 드는 썸네일을 발견할 때마다 폴더에 저장하기만 하면 된다. 분류와 정리는 Cowork가 알아서 처리한다.

**주의사항**: 예약 작업이 실행되려면 **Claude 데스크톱 앱이 열려 있어야 한다**. 앱이 닫혀 있을 경우, 다음 번에 앱을 열 때 해당 작업이 자동으로 시작된다.

## 7. 실제 활용 사례 2: 캘린더 기반 회의 준비 자동화

이 예시는 Cowork의 진정한 능력을 보여준다. 단 하나의 간결한 프롬프트로 웹 리서치, 문서 생성, 이메일 초안 작성까지 모두 자동으로 처리한다.

### 프롬프트

> *"내 캘린더에서 회의 일정을 확인하고 준비를 도와주세요. 각 회의에는 제가 만들어야 할 문서와 해야 할 작업에 대한 메모가 있습니다. 메모를 읽고 모든 지시사항을 따르세요."*

### 캘린더 메모의 내용

사용자의 캘린더에는 'Anthropic 인수 회의'가 등록되어 있었고, 메모에는 세 가지 산출물이 명시되어 있었다.

1. **1페이지 기업 브리프**: Anthropic의 주요 자산, 기업 가치, 리더십 팀, 전략적 적합성을 담은 인수 대상 보고서
2. **PowerPoint 프레젠테이션**: "Futurepedia의 Anthropic 전략적 인수"라는 제목의 슬라이드 - 인수 타당성, 브랜드 시너지, 인수 후 비전, 다음 단계 포함
3. **Gmail 초안 이메일**: Dario Amodei(Anthropic CEO)에게 보내는 전략적 거래 제안 이메일 (적극적인 톤으로)

### 처리 과정

Cowork는 단계별로 다음을 수행했다.

1. **캘린더 연동**: Google Calendar에서 회의 일정과 메모를 자동으로 읽음
2. **웹 리서치**: Anthropic에 관한 다수의 웹사이트를 방문하여 필요한 정보 수집
3. **병렬 작업 실행**: 기업 브리프 작성과 PowerPoint 제작을 동시에 시작
4. **이메일 초안 작성**: Gmail 연동을 통해 직접 임시보관함에 저장
5. **최종 검증**: 모든 산출물이 요구사항을 충족하는지 확인

### 결과물

- **기업 브리프**: 정확히 1페이지, 깔끔하고 스캔하기 좋은 포맷. "전략적 적합성" 섹션에는 YouTube 기반 AI 채널이 기반 모델 레이어를 소유함으로써 얻는 수직 통합 기회 등이 포함
- **PowerPoint**: Keynote에서 바로 열리는 완전히 편집 가능한 실제 파일. 각 슬라이드는 논리적으로 구성되어 있으며 실제 데이터와 설득력 있는 내러티브 포함
- **Gmail 초안**: 공격적이고 긴박감 있는 톤으로 작성된 이메일. "테이블에 앉지 않으면 메뉴판에 올라가게 된다"는 문구 포함

## 8. 모델 선택과 토큰 관리

### 모델 선택

Cowork에서 작업 시작 전 사용할 모델을 선택할 수 있다.

- **Sonnet + 생각 모드(Thinking Mode)**: 대부분의 작업에 충분하고 권장되는 선택
- **Opus**: 매우 복잡한 작업에는 업그레이드 가능. 가장 강력하지만 토큰 소모가 훨씬 빠름

### 토큰 한도 주의사항

세션당 토큰 한도가 있다. Pro 플랜의 경우 특히 길고 무거운 작업을 반복하면 한도에 도달할 수 있으며, 이 경우 몇 시간을 기다려야 한다. Claude Code를 함께 사용할 경우 Opus 사용량이 많아지므로 더욱 주의가 필요하다.

## 9. 작업 중 권한 및 확인 요청

Cowork는 실제 파일을 다루기 때문에 **안전 장치**가 마련되어 있다.

- 작업 시작 시 파일 접근 권한 승인 요청
- 가끔 작업 중간에 멈추고 확인을 기다리는 경우도 있음 (드문 경우)
- 명확하지 않은 부분은 추가 질문

이로 인해 자리를 비웠을 때 진행이 멈춰 있는 경우가 있을 수 있다. 처음에는 확인이 필요하지만 이후에는 대부분 자율적으로 진행된다.

## 10. 실용적인 팁

### 폴더 복제 습관 만들기

Cowork를 사용할 때는 반드시 **원본 폴더를 복제한 뒤** 복제본을 대상으로 작업하도록 지정해야 한다. 작업 결과가 마음에 들지 않거나 오류가 발생해도 원본 데이터는 안전하게 보존된다.

### Chat 탭과 Cowork 탭 함께 활용하기

복잡한 작업을 위한 상세한 프롬프트가 필요할 때는 먼저 **Chat 탭에서 Claude에게 프롬프트 확장 및 구조화를 요청**한 뒤, 그 결과를 Cowork 탭에 붙여넣는 방법이 효과적이다.

### 소규모 테스트 먼저

대량의 파일을 처리하기 전에 소수의 파일로 먼저 테스트해보면 원하는 결과를 얻을 수 있는지 미리 확인할 수 있다.

## 11. 현재 한계점

현재 Claude Cowork는 **리서치 프리뷰(Research Preview)** 단계이므로 몇 가지 제약이 있다.

### 세션 간 메모리 부재

현재 Cowork는 **세션 간 메모리**가 없다. 이전 세션에서 무엇을 했는지 기억하지 못한다. 이에 대한 현실적인 해결책은 **폴더 구조와 로그 파일 활용**이다. 즉, 매 세션마다 업데이트되는 작업 로그 파일을 만들어 관리하도록 Cowork에 지시하면 된다. 구체적인 방법은 사용 사례에 따라 다르지만, 이것이 현재 시점에서의 표준적인 해결 방안이다.

### 그 외 알려진 제한

- 가끔 버그가 발생할 수 있음 (프리뷰 단계임을 감안)
- 예약 작업은 앱이 열려 있어야만 실행됨
- Pro 플랜에서는 집중적인 작업 시 토큰 한도에 도달할 수 있음

---

## 12. 결론: 업무 위임의 새로운 패러다임

Claude Cowork는 단순히 더 나은 챗봇이 아니다. 이것은 AI와 인간이 협력하는 방식 자체를 바꿔놓는 도구다. 스크린샷 정리처럼 사소하지만 시간이 걸리는 작업부터, 캘린더를 읽고 웹에서 정보를 수집하여 전문 문서를 만들고 이메일 초안을 작성하는 복잡한 워크플로우까지, 단 하나의 프롬프트로 처리한다.

모든 회의에 맞춤형 준비 자료, 모든 반복 작업의 자동화, 모든 복잡한 멀티스텝 프로젝트의 자율 실행. 이것이 Claude Cowork가 가리키는 미래다. AI가 질문에 답하는 도구에서, 실제로 일을 해내는 동료로 진화하고 있다.
