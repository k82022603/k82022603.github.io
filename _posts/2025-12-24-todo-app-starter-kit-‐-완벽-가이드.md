## 📦 패키지 내용물

이 스타터 킷은 바이브코딩 원칙을 실습할 수 있는 완전한 프로젝트 구성을 포함합니다.

### 파일 구조

```
todo-app-starter-kit/
├── CLAUDE.md                          # Claude Code 시스템 프롬프트 ⭐
├── PLAN.md                            # 단계별 구현 계획 ⭐
├── README.md                          # 프로젝트 설명서
├── todo-app-vibe-coding-guide.md      # 상세 구현 가이드 ⭐
└── .claude/
    └── commands/                      # 커스텀 슬래시 커맨드
        ├── implement-feature.md       # 기능 구현 템플릿
        └── fix-bug.md                 # 버그 수정 템플릿
```

⭐ = 반드시 읽어야 할 핵심 파일

## 📄 각 파일의 역할

### 1. CLAUDE.md - "Claude의 두뇌"

**용도**: Claude Code가 프로젝트를 이해하는 데 필요한 모든 컨텍스트  
**언제 사용**: 항상! Claude Code는 매 대화마다 이 파일을 자동으로 읽습니다

**포함 내용**:
- ✅ 프로젝트 기술 스택
- ✅ 폴더 구조 및 파일 규칙
- ✅ 코딩 표준 (TypeScript, React 패턴)
- ✅ 세 가지 바이브코딩 원칙
- ✅ 일반적인 실수 방지 가이드
- ✅ 테스트 가이드라인

**중요성**: 
- 이 파일이 없으면 Claude는 매번 프로젝트 구조를 다시 설명받아야 합니다
- 이 파일이 있으면 Claude는 일관된 코드 스타일을 유지합니다
- 팀 전체가 같은 규칙을 따르도록 합니다

**사용 예시**:
```bash
# Claude Code에서
You: 새로운 컴포넌트를 만들고 싶어
Claude: CLAUDE.md를 확인하니 컴포넌트는 src/components/에 
        PascalCase로 만들고, Props interface를 
        같은 파일에 정의하는 규칙이네요...
```

### 2. PLAN.md - "로드맵"

**용도**: 프로젝트를 처음부터 끝까지 완성하는 단계별 계획  
**언제 사용**: 프로젝트를 시작하고 각 단계를 진행할 때

**포함 내용**:
- ✅ 7개 Phase로 구성된 구현 순서
- ✅ 각 Phase별 예상 소요 시간
- ✅ 각 단계의 검증 체크리스트
- ✅ Claude Code에 제공할 구체적인 프롬프트
- ✅ 테스트 시나리오

**중요성**:
- 무엇을 먼저 만들어야 할지 고민할 필요 없음
- 각 단계가 독립적이라 중단했다가 이어서 할 수 있음
- 프롬프트가 이미 작성되어 있어 복사만 하면 됨

**사용 예시**:
```bash
# 프로젝트 시작할 때
You: @PLAN.md Phase 1부터 시작하자

# 진행 중일 때
You: @PLAN.md 지금 어느 단계야? 다음은?

# 각 Phase의 프롬프트를 복사해서 사용
You: [Phase 2의 프롬프트 복사 & 붙여넣기]
```

### 3. todo-app-vibe-coding-guide.md - "교과서"

**용도**: 전체 프로젝트의 상세 설명서이자 학습 자료  
**언제 사용**: 프로젝트 시작 전 전체 이해가 필요할 때, 막혔을 때 참고

**포함 내용**:
- ✅ 바이브코딩 3원칙 상세 설명
- ✅ 각 기능의 요구사항 분해 예시
- ✅ 실패 기준 정의 예시
- ✅ 프롬프트 작성 예시
- ✅ 검증 방법

**중요성**:
- PLAN.md는 "무엇을", 이 파일은 "왜, 어떻게"를 설명
- 바이브코딩 원칙을 실제로 적용하는 방법 학습
- 막혔을 때 돌아와서 읽을 수 있는 참고서

**사용 예시**:
```bash
# 개념을 이해하고 싶을 때
"요구사항 쪼개기가 정확히 뭐지?" → 가이드의 2부 참고

# 프롬프트 작성이 막힐 때
"어떻게 명확하게 요청해야 하지?" → 가이드의 3부 참고

# 실패 기준을 정하고 싶을 때
"무엇을 실패로 봐야 하지?" → 가이드의 4부 참고
```

### 4. .claude/commands/ - "단축키"

**용도**: 반복적인 작업을 템플릿화한 커스텀 커맨드  
**언제 사용**: 새 기능 추가나 버그 수정 시

#### implement-feature.md
```bash
# 사용법
claude> /implement-feature "할 일 편집 기능"

# 그러면 자동으로:
# 1. 기능을 작은 단위로 쪼개라고 요청
# 2. 요구사항을 명확히 하라고 요청
# 3. 실패 기준을 정의하라고 요청
# 4. 구현을 시작
# 5. 검증 체크리스트 제공
```

#### fix-bug.md
```bash
# 사용법
claude> /fix-bug "완료 버튼이 작동하지 않음"

# 그러면 자동으로:
# 1. 버그 재현 단계 확인
# 2. 원인 분석
# 3. 테스트 케이스 작성
# 4. 수정
# 5. 검증
# 6. 예방책 제시
```

### 5. README.md - "프로젝트 소개서"

**용도**: 프로젝트 개요와 빠른 시작 가이드  
**언제 사용**: 프로젝트를 처음 접할 때, 다른 사람에게 공유할 때

**포함 내용**:
- ✅ 프로젝트 목적
- ✅ 설치 방법
- ✅ 실행 방법
- ✅ 기능 목록
- ✅ 일반적인 문제 해결

## 🚀 프로젝트 시작하기

### 준비물 체크리스트

- [ ] Node.js 18 이상 설치됨
- [ ] Claude Code CLI 설치됨 ([설치 가이드](https://claude.com/code))
- [ ] VS Code (선택사항이지만 권장)
- [ ] 터미널 접근 가능

### Step 1: 프로젝트 준비

```bash
# 1. 프로젝트 폴더 생성
mkdir todo-app
cd todo-app

# 2. 스타터 킷의 모든 파일을 여기에 복사
# CLAUDE.md, PLAN.md, README.md 등

# 3. .claude 폴더도 복사 (숨김 폴더 주의!)
```

### Step 2: 파일 읽기 순서

**첫 번째 세션 전에 꼭 읽어야 할 순서**:

1. **README.md** (5분) - 프로젝트가 뭔지 이해
2. **PLAN.md의 개요** (10분) - 전체 흐름 파악
3. **CLAUDE.md 훑어보기** (5분) - 규칙 확인
4. **todo-app-vibe-coding-guide.md의 서론** (10분) - 원칙 이해

총 30분 투자로 프로젝트 전체를 이해할 수 있습니다.

### Step 3: Claude Code 시작

```bash
# 프로젝트 폴더에서 Claude Code 시작
claude

# 첫 번째 명령: 프로젝트 초기화
You: /init

# Claude가 CLAUDE.md를 자동 생성하려고 하면
You: 이미 CLAUDE.md가 있어. 그걸 사용해줘.

# 이제 PLAN.md를 참조해서 Phase 1 시작
You: @PLAN.md Phase 1의 프롬프트대로 프로젝트를 설정해줘.
```

### Step 4: 단계별 진행

```bash
# Phase 1 완료 후
You: Phase 1이 완료됐어. 검증 체크리스트 확인해줘.

# 검증 완료 후
You: @PLAN.md Phase 2로 넘어가자.

# 각 Phase마다 이 패턴 반복
```

## 💡 효과적인 사용 팁

### Tip 1: CLAUDE.md 활용

```bash
# 규칙이 궁금할 때
You: @CLAUDE.md 컴포넌트 만들 때 규칙이 뭐야?

# 에러 처리 방법이 궁금할 때
You: @CLAUDE.md 에러는 어떻게 처리해야 해?

# 테스트 작성 방법이 궁금할 때
You: @CLAUDE.md 테스트는 어떻게 구조화해?
```

### Tip 2: PLAN.md 활용

```bash
# 현재 진행 상황 확인
You: @PLAN.md 지금까지 뭘 했고 다음은 뭐야?

# 특정 Phase 건너뛰고 싶을 때
You: @PLAN.md Phase 3는 건너뛰고 Phase 4부터 할 수 있어?

# 막혔을 때
You: @PLAN.md Phase 2에서 막혔어. 검증 체크리스트 다시 확인해줘.
```

### Tip 3: 슬래시 커맨드 활용

```bash
# 새 기능 추가할 때
You: /implement-feature "할 일 수정 기능 추가"

# 버그 발견했을 때
You: /fix-bug "삭제 버튼 클릭 시 에러 발생"

# 이렇게 하면 3원칙을 자동으로 적용하게 됨!
```

## ⚠️ 자주하는 실수

### 실수 1: CLAUDE.md를 안 읽음
```
❌ Claude에게 "컴포넌트 만들어줘"만 반복
✅ 먼저 @CLAUDE.md로 프로젝트 규칙 확인
```

### 실수 2: 한 번에 너무 많이 하려고 함
```
❌ "전체 앱을 한 번에 만들어줘"
✅ PLAN.md를 따라 Phase별로 진행
```

### 실수 3: 검증을 건너뜀
```
❌ 코드 생성되자마자 다음 단계로
✅ 각 Phase의 검증 체크리스트 확인
```

### 실수 4: 3원칙을 안 지킴
```
❌ 애매한 요청: "로그인 기능 만들어줘"
✅ 명확한 요청:
   1. 쪼개기: 입력폼 + 검증 + API 호출 + 에러처리
   2. 명확화: 각각의 입출력, 제약사항 정의
   3. 실패 기준: 빈 입력, 잘못된 형식 등 정의
```

### 실수 5: 파일을 안 읽고 질문만 함
```
❌ "이거 어떻게 해?"
✅ 먼저 해당 파일 확인 후 구체적으로 질문
```

## 📚 문제 해결 가이드

### "어디서부터 시작해야 할지 모르겠어요"
→ **해결**: README.md 읽고 → PLAN.md Phase 1 시작

### "Claude가 이상한 코드를 생성해요"
→ **해결**: @CLAUDE.md 확인 → 규칙 명시적으로 상기

### "중간에 막혔어요"
→ **해결**: @PLAN.md 현재 Phase 검증 체크리스트 확인

### "버그가 생겼어요"
→ **해결**: /fix-bug "[구체적인 증상]" 사용

### "새 기능을 추가하고 싶어요"
→ **해결**: /implement-feature "[기능 이름]" 사용

### "다 만들었는데 뭘 더 할까요?"
→ **해결**: PLAN.md "Next Steps" 섹션 참고
